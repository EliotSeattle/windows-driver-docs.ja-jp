# <a name="feature-description"></a>機能の説明


クラッシュ ダンプのドライバーでは、フィルター ドライバーをクラッシュ ダンプのスタックに追加する新しいインターフェイスを提供します。 このインターフェイスは、任意の標準のフィルター ドライバー モデルに従っていません。 代わりに、クラッシュ ダンプのスタックにドライバーを読み込むために使用する必要がありますが、専用のインターフェイスを公開します。

レジストリ キーで、サービス名を追加することでは、クラッシュ ダンプのフィルター ドライバーをダンプ スタックの一部行われたことができます。 クラッシュ ダンプのドライバーは、ダンプ スタックのドライバーが読み込まれるときに、これらのドライバーを読み込みます。 フィルター ドライバーは、休止状態とクラッシュ ダンプのスタックの一部として読み込まれます。 これらのフィルター ドライバーは、休止状態やクラッシュ ダンプの中に呼び出される定義済みのコールバック ルーチンの一覧を指定する必要があります。

フィルターのモデルでは、休止状態またはクラッシュ ダンプのスタックにのみ読み込まれるフィルター ドライバーは許可されません。 サービス名がフィルター ドライバーの一覧に追加されると、一覧が両スタックで読み込まれます。 コールバック ルーチンが呼び出されると、コールバックの理由を示すために、パラメーターが渡されます。 このパラメーターに基づき、どのようなフィルター ドライバーを決定できます。
