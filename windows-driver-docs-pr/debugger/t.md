# <a name="t"></a>T


<span id="target"></span><span id="TARGET"></span>**ターゲット**  
対象のアプリケーション、対象のコンピューターまたはダンプ ターゲットについてはします。

呼ば、*デバッグ対象*します。

<span id="target_application"></span><span id="TARGET_APPLICATION"></span>**ターゲット アプリケーション**  
アプリケーションは、ユーザー モードでデバッグ中です。

<span id="target_computer"></span><span id="TARGET_COMPUTER"></span>**対象のコンピュータ**  
カーネル モードでは、デバッグ中のコンピューター。

<span id="target_id"></span><span id="TARGET_ID"></span>**ターゲット ID**  
ターゲットの一意の識別子。

<span id="target_process"></span><span id="TARGET_PROCESS"></span>**ターゲット プロセス**  
ユーザー モードでデバッグ対象アプリケーションの一部であるオペレーティング システム プロセスです。

カーネル モードでのデバッグ、カーネルを表す仮想プロセス。

<span id="target_thread"></span><span id="TARGET_THREAD"></span>**対象のスレッド**  
ユーザー モードでデバッグ ターゲットのアプリケーションでのオペレーティング システム スレッドです。

カーネル モードでのデバッグ、プロセッサを表す仮想スレッド。

<span id="thread_context"></span><span id="THREAD_CONTEXT"></span>**スレッド コンテキスト**  
スレッドを切り替えるときに、Windows によって保持状態です。 レジスタのコンテキストがスレッド コンテキストではなくの一部であるいくつか余分なカーネル専用のプロセッサの状態があることを除き、レジスタのコンテキストに似ています。 この余分な状態は、カーネル モードのデバッグ中にレジスタとして使用できます。

詳細については、スコープとシンボルのグループを参照してください。

<span id="transport_layer"></span><span id="TRANSPORT_LAYER"></span>**トランスポート層**  
これは、リモート デバッグ中に、ホストとターゲットのコンピューター間の通信を制御します。

<span id="trap"></span><span id="TRAP"></span>**トラップ**  
バグの確認を参照してください。

<span id="type"></span><span id="TYPE"></span>**型**  
記号の型を参照してください。
