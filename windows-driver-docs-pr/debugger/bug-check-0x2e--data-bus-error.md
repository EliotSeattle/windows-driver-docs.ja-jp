# <a name="bug-check-0x2e-databuserror"></a>バグ チェック 0x2E:データ\_BUS\_エラー

<a name="cause"></a>原因
-----

このエラーはほとんど構成の問題、ハードウェア障害、または互換性のないハードウェア - ハードウェアの問題が原因です。

このエラーを引き起こす可能性のある最も一般的なハードウェアの問題は欠陥の RAM、レベル 2 (L2) の RAM のキャッシュ エラー、またはビデオ RAM エラー。 ハード ディスクの破損もこのエラーがあります。

このバグ チェックは、デバイス ドライバーが、0x8 内のアドレスにアクセスしようとしたときにも発生する*xxxxxxx*範囲 (つまり、物理アドレスのマッピングがない) に存在しません。

<a name="resolution"></a>解決方法
----------

**ハードウェアの問題を解決するには。** ハードウェアは、システムに最近追加されたの場合は、エラーが再発するかどうかに表示することを削除します。

既存のハードウェアが失敗した場合は、削除または障害のあるコンポーネントの交換します。 失敗したハードウェア コンポーネントが決定するシステムの製造元から提供されたハードウェア診断を実行してください。 詳細については、これらの手順は、コンピューターの製造元のマニュアルを参照してください。 コンピューターのすべてのアダプター カードが正しく取り付けられていることを確認します。 インクの削除や、電気連絡先処理、サプライの電器店でご利用いただけますを使用して、アダプター カードの連絡先がクリーンであることを確認します。

新しくインストールされたシステムで問題が発生する場合は、SCSI コント ローラーまたはネットワーク カードは、BIOS の更新プログラムの可用性を確認します。 この種の更新プログラムは通常、Web サイトまたはハードウェアの製造元の掲示板システム (BBS です)。

新しいまたは更新されたデバイス ドライバーをインストールした後、エラーが発生した場合、ドライバーを削除または置換する必要があります。 場合に、起動中にこのような状況では、エラーが発生したシステム パーティションが NTFS でフォーマットされた、セーフ モードを使用して、名前を変更または障害のあるドライバーを削除できる可能性があります。

セーフ モードでシステムの起動プロセスの一部としてドライバーを使用する場合は、ファイルにアクセスするには、回復コンソールを使用してコンピューターを起動する必要があります。
