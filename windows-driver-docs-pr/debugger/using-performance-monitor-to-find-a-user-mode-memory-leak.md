# <a name="using-performance-monitor-to-find-a-user-mode-memory-leak"></a>パフォーマンス モニターを使用したユーザーモード メモリ リークの検出


ユーザー モードのメモリ リークが不明などのプロセスが原因があると思われる場合は、個々 のプロセスのメモリ使用量を測定するパフォーマンス モニターを使用できます。

パフォーマンス モニターを起動します。 次のカウンターを追加します。

-   **プロセス**--&gt;**Private Bytes** (の各プロセスを確認する)

-   **プロセス**--&gt;**仮想バイト**(の各プロセスでは、調査する)

更新時間を時間の経過と共に、リークのグラフをキャプチャする 600 秒に変更します。 データを後で調査用にファイルに記録する場合があります。

**Private Bytes**カウンターは、その他のプロセスと共有メモリを除く、割り当てプロセス メモリの総量を示します。 **仮想バイト**カウンターは、プロセスを使用している仮想アドレス空間の現在のサイズを示します。

増加がデータ ファイルにいくつかのメモリ リークが表示されます (割り当てられているプライベート バイト単位)。 仮想アドレス空間の増加がスライド ショーの他のメモリ リークします。

メモリをリークしているプロセスを確認した後は、UMDH ツールを使用して、問題なのでは、特定のルーチンを決定します。 詳細については、[ユーザー モード メモリ リークの検出に使用する UMDH](using-umdh-to-find-a-user-mode-memory-leak.md)を参照してください。
