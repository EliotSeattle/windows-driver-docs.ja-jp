# <a name="bug-check-0xe0-acpibiosfatalerror"></a>バグ チェック 0xE0:ACPI\_BIOS\_FATAL\_エラー


ACPI\_BIOS\_FATAL\_エラーのバグ チェックが 0x000000E0 の値を持ちます。 これは、コンピューターのコンポーネントの 1 つが障害のあることを示します。

> [!IMPORTANT]
> このトピックはプログラマーを対象としています。 コンピューターを使用しているときに、エラー コードがブルー スクリーンが受信した顧客の場合を参照してください。[トラブルシューティング ブルー スクリーン エラー](https://www.windows.com/stopcode)します。


## <a name="acpibiosfatalerror-parameters"></a>ACPI\_BIOS\_FATAL\_エラー パラメーター


このバグ チェックのパラメーターは、Windows ではなく、BIOS によって発行されます。 また、ハードウェア ベンダーによってのみ解釈できます。

<a name="cause"></a>原因
-----

コンピューターの BIOS がシステム内のコンポーネントは、Windows が動作するための方法がないため問題のあることを報告しました。 BIOS では、バグ チェックを発行する代替がないことを示すです。

<a name="resolution"></a>解決方法
----------

診断のディスクまたはコンピューターに含まれていたツールを実行しているコンポーネントが問題のあるを指定できます。

場合、このツールはありませんは、システムのベンダーにお問い合わせくださいし、には、このエラー メッセージを報告する必要があります。 このハードウェアの問題を解決するためできなきます。 これにより、動作する Windows です。

Microsoft は、このエラーに対処できません。 ハードウェアの製造元は分析を修飾します。
