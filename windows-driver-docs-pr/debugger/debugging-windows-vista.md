# <a name="debugging-windows-vista"></a>Windows Vista のデバッグ


WinDbg を使用して、Windows Vista をデバッグするに含まれている Windows 7 のデバッグ ツールの Windows パッケージを取得、 [Microsoft Windows ソフトウェア開発キット (SDK) の Windows 7 および .NET Framework 4.0](https://go.microsoft.com/fwlink/p/?LinkId=320327)します。

SDK をインストールし、インストール中、次のように選択します。 のみのデバッグ ツールの Windows をダウンロードする場合、**ツールを Windows のデバッグ**ボックスとその他のすべてのボックスをオフにします。

**注**  SDK をインストールする前に、Microsoft Visual C 2010 再頒布可能コンポーネントをアンインストールする必要があります。 詳細については、[、Microsoft サポート web サイト](https://support.microsoft.com/kb/2717426)を参照してください。

 

## <a name="span-iddebuggingtoolswindbgkdcdbntsdforwindowswindowsvistaspandebugging-tools-windbg-kd-cdb-ntsd-for-windows-vista"></a><span id="DEBUGGING_TOOLS__WINDBG__KD__CDB__NTSD__FOR_WINDOWS_WINDOWS_VISTA"></span>Windows Vista 用デバッグ ツール (WinDbg、KD、CDB、NTSD)


X86 ベースまたは x64 ベースのプロセッサで、Windows 7 のデバッグ ツールの Windows を実行し、x86 ベースで実行されているコードまたは x64 ベース プロセッサをデバッグすることができます。 デバッガーとデバッグ対象のコードは、同じコンピューター上で実行されることも、別々のコンピューターで実行されることもあります。 いずれの場合も、デバッガーが実行されるコンピューターは "*ホスト コンピューター*" と呼ばれ、デバッグ対象のコンピューターは "*ターゲット コンピューター*" と呼ばれます。 ターゲット コンピューターがこれらのオペレーティング システムのいずれかを実行している場合は、Windows 7 のデバッグ ツールの Windows を使用します。
