# <a name="symbols"></a>シンボル


A*シンボル*はデータやコード モジュールに表示されるソース ファイルからの名前付き単位です。 シンボルに関する情報は、名前、型 (該当する) 場合、アドレスまたはレジスタ格納場所および任意の親または子のシンボルを含めることができます。 シンボルの例には、変数 (ローカルおよびグローバル)、関数、およびモジュールへのエントリ ポイントが含まれます。

シンボル情報は、データと、ターゲット内のコードの解釈に役立つ、エンジンによって使用されます。 この情報により、エンジンは、名前またはメモリ内の場所でシンボルの検索し、シンボルの説明を入力します。

エンジンは、ローカル ファイル システム上にあるか、シンボル サーバーから読み込まれたシンボル ファイルからシンボルについての情報を取得します。 シンボル サーバーを使用する場合、エンジンは自動的に使用されているターゲット内のモジュールと一致するシンボル ファイルの正しいバージョンを使用します。 対応するモジュールが読み込まれる、または必要に応じて読み込むことができるには、常に、シンボル ファイルを読み込むことができます。

**注**  多くの場合、コンパイラの最適化、正確な情報に含めないシンボル ファイル。 変数の場所としていくつかの変数の値を誤って解釈するエンジンが生じるまたは有効期間正しく説明はない、メモリの問題の部分を見てまたは変数の値は、ライブ配信不能 (またはその逆) ときに、エンジンの原因です。 最適化コンパイラの実行順序を変更するか関数をいくつかの部分に分割することもできます。 最適化されていないコードをデバッグするときに、最適な結果が通常取得されます。
