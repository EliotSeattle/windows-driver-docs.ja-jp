# <a name="breakpoints"></a>ブレークポイント


[デバッガー エンジン](introduction.md#debugger-engine)を作成し、ターゲット内のブレークポイントを監視します。

エンジンがターゲットに挿入できる、ブレークポイントの 2 種類があります。 ソフトウェアのブレークポイントとブレークポイントのプロセッサ。

-   *ソフトウェアのブレークポイント*プロセッサの命令にブレークポイントの場所を変更することで、ターゲットのコードに挿入されます。 デバッガー エンジンにはこのようなブレークポイントの追跡読み取りと書き込みのメモリがその場所でクライアントに表示されません。 ターゲットが変更された命令を実行するときは、ソフトウェアのブレークポイントがトリガーされます。

-   *プロセッサのブレークポイント*デバッガー エンジンによって、ターゲットのプロセッサに挿入されます。 プロセッサ ブレークポイントでトリガーできますさまざまなアクションは、たとえば、(ブレークポイント)、ソフトウェアのような場所に、命令の実行の読み取りまたはブレークポイントの位置にメモリを記述します。 プロセッサのブレークポイントのサポートは、ターゲットのコンピューターのプロセッサに依存します。

明示的なアドレス、アドレスに評価される式または式を将来の時刻に、アドレスに評価される場合があります、ブレークポイントのアドレスを指定できます。 最後の場合、モジュールが読み込まれたまたは、ターゲットでアンロードされるたびに、エンジンは、式の再評価してしよう、アドレスを調べる場合、ブレークポイントの挿入これにより、読み込まれる前に、モジュール内でブレークポイントを設定します。

パラメーターの数は、その動作を制御するブレークポイントを関連付けることができます。

-   ブレークポイントのターゲットの特定のスレッドに関連付けることができ、、そのスレッドによってのみトリガーされます。

-   ブレークポイントは、それに関連付けられているデバッガー コマンドを持つことができます。これらのコマンドは、ブレークポイントがトリガーされたときに自動的に実行されます。

-   ブレークポイントは、ターゲットが経過するまで、指定した回数だけ、非アクティブとしてフラグが付けられますことができます。

-   ブレークポイントは自動的に削除できます、トリガーは最初の時刻。
