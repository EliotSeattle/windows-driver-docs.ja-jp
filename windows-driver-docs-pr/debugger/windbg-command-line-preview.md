# <a name="windbg-preview---command-line-startup-options"></a>WinDbg プレビュー - コマンド ライン スタートアップ オプション

## <a name="starting-windbg-preview"></a>WinDbg のプレビューを開始

WinDbg のプレビューをインストールした後 WinDbgX.exe はディレクトリの任意の場所から実行できるようにします。 


## <a name="command-line-startup-options"></a>コマンド ライン スタートアップ オプション

```dbgsyntax
WinDbgX [options]
```

この次の表では、使用できるコマンド ライン オプションをまとめたものです。

**全般オプション**


|     オプション      |                                                                          説明                                                                          |
|-----------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------|
| -c  *"command"* | デバッガーをアタッチした後は、コマンドラインを実行します。 このコマンドは、引用符で囲む必要があります。 複数のコマンドは、セミコロンで区切ることができます。 |
|       -v        |                                                            デバッガーで詳細な出力を有効にします。                                                            |
|   -T *Title*    |                                                                    ウィンドウのタイトルを設定します。                                                                     |
| -logo *LogFile*  |                               開いているログに記録します。 ログ ファイルにログ情報を開始します。 ファイルが存在する場合は上書きされます。                                |
| -loga *LogFile*  |                              ログを追加します。 ログ ファイルにログ情報を開始します。 ファイルが存在する場合に追加されます。                               |
| -e *EventHandle* |                                         ターゲットの次の例外の後に指定されたハンドル イベントを通知します。                                         |
|       -?        |                                                           使用可能なコマンドの概要を表示します。                                                           |

**カーネルのオプション**


|       オプション       |                                                                      説明                                                                      |
|--------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------|
| -k \[ConnectType\] | カーネル デバッグ セッションを開始します。  場合 **-k**せず使用*ConnectType*オプションの後に、コマンドラインで最終的なエントリがあります。 |
|        -kqm        |                                                               KD をサイレント モードで起動します。                                                                |
|        -kl         |                                        カーネル デバッガーと同じコンピューター上のセッションのデバッグを開始します。                                         |
| -kx *ExdiOptions*  |                                                カーネル デバッグ EXDI ドライバーを使用してセッションを開始します。                                                |
|         -d         |                        再起動後、デバッガーは、カーネル モジュールが読み込まれるとすぐに目的のコンピュータに中断されます。                         |

**ユーザー モードのオプション**

オプション | 説明
|------ | -----------|
-o | ターゲット アプリケーション (子プロセス) で起動されたすべてのプロセスをデバッグします。 
-g | ターゲット アプリケーションでは、最初のブレークポイントを無視します。 
-G |ターゲット アプリケーションで最終的なブレークポイントを無視します。 
-pv | デバッガーがターゲット プロセスに noninvasively アタッチする必要がありますを指定します。
-hd | デバッグ ヒープを使用しないことを指定します。
-cimp | 作成されたすべてのプロセスがクライアントから特定のユーザー コマンド ライン文字列ではなくサーバーによって、暗黙的なコマンド ライン セットを使用するを指定します。

**シンボルのオプション**

オプション | 説明
|------ | -----------|
-y *SymbolPath* | 使用するシンボル パスを指定します。 複数のパスをセミコロンで区切ります (**;**)。 パスにスペースが含まれている場合は、引用符で囲む必要があります。 詳細については、およびこのパスを変更するには、他の方法については、「[シンボル パス](symbol-path.md)します。

-i *ImagePath* | 使用するイメージの検索パスを設定します。
-sdce | 原因のデバッガーがシンボルの読み込み中に「ファイル アクセス エラー」メッセージを表示します。 
-ses | 原因のデバッガーがすべてのシンボル ファイルの厳密な評価を実行し、問題のある任意の記号を無視します。

-snc | 原因のデバッガーが C++ の変換をオフにします。
-snul | 非修飾名のシンボルの自動読み込みを無効にします。
-sup | すべてのシンボルの検索中に、パブリック シンボル テーブルを検索するシンボル ハンドラーと、します。
-sflags| 一度にすべてのシンボルのハンドラーのオプションを設定します。
