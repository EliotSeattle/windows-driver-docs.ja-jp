# <a name="windbg-preview---command-line-startup-options"></a>WinDbg プレビュー - コマンド ライン スタートアップ オプション

![WinDbg プレビューの小さなロゴ](images/windbgx-preview-logo.png)

## <a name="starting-windbg-preview"></a>WinDbg プレビューを開始しています

WinDbg Preview をインストールすると、任意のディレクトリの場所から WinDbgX .exe を実行できるようになります。

## <a name="command-line-startup-options"></a>コマンドラインスタートアップオプション

```dbgsyntax
WinDbgX [options]
```

次の表は、使用可能なコマンドラインオプションの概要を示しています。

**全般オプション**

|     オプション      |                                                                          説明                                                                          |
|-----------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------|
| -c *"command"*  | デバッガーがアタッチされた後にコマンドラインを実行します。 このコマンドは引用符で囲む必要があります。 複数のコマンドは、セミコロンで区切ることができます。 |
| -v               | デバッガーで詳細出力を有効にします。                                                            |
| -T*タイトル*       | ウィンドウのタイトルを設定します。                                                                     |
| -logo *LogFile*  | ログを開きます。 ログファイルへの情報のログ記録を開始します。 ファイルが存在する場合は上書きされます。                                |
| -loga*ログファイル*  | ログの追加。 ログファイルへの情報のログ記録を開始します。 ファイルが存在する場合は、に追加されます。                               |
| -e *EventHandle* | ターゲットの次の例外の後に、指定されたハンドルを使用してイベントを通知します。                                         |
|       -?         |  使用可能なコマンドの概要が表示されます。                                                           |

**カーネルオプション**

|       オプション       |                                                                      説明                                                                      |
|--------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------|
| -k \[ConnectType\] | カーネルデバッグセッションを開始します。  **-K**が、その後に*connecttype*オプションを指定せずに使用されている場合は、コマンドラインの最後のエントリである必要があります。 |
|        -kqm        | 非表示モードで KD を開始します。                                                                |
|        -kl ダイバージェンス         | デバッガーと同じコンピューター上でカーネルデバッグセッションを開始します。                                         |
| -kx *Exdioptions*  | EXDI ドライバーを使用して、カーネルデバッグセッションを開始します。                                                |
|         -d         | 再起動後、カーネルモジュールが読み込まれるとすぐに、デバッガーはターゲットコンピューターを中断します。                         |

**ユーザーモードのオプション**

オプション | 説明
|------ | -----------|
-o | ターゲットアプリケーション (子プロセス) によって起動されたすべてのプロセスをデバッグします。 
-g | ターゲットアプリケーションの最初のブレークポイントを無視します。 
-G |ターゲットアプリケーションの最後のブレークポイントを無視します。 
-pv | デバッガーをターゲットプロセス noninvasively にアタッチするように指定します。
-hd | デバッグヒープを使用しないことを指定します。
-cimp | 作成されたすべてのプロセスが、クライアントからのユーザー指定のコマンドライン文字列ではなく、サーバーによって設定された暗黙的なコマンドラインを使用することを指定します。

**ターゲットオプション**

| オプション                     |                 説明                                                                                                                                  |
|----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| -リモート*Clienttransport*  | 既に実行されているデバッグサーバーに接続します。 使用可能な*Clienttransport*値の詳細については、「[デバッグクライアントのアクティブ化](activating-a-debugging-client.md)」を参照してください。 このパラメーターを使用する場合は、コマンドラインの最初のパラメーターである必要があります。 |
| -サーバー *Servertransport*  | 他のデバッガーからアクセスできるデバッグサーバーを作成します。 使用可能な*Servertransport*値の詳細については、「[デバッグサーバーのアクティブ化](activating-a-debugging-server.md)」を参照してください。              |
| -premote *Smartclienttransport*   | スマートクライアントを作成し、既に実行されているプロセスサーバーに接続します。 SmartClientTransport の有効な値の説明については、「[スマートクライアントのアクティブ化](activating-a-smart-client.md)」を参照してください。 |
| -p *PID*               | デバッグする10進プロセス ID を指定します。                                                                                                              |
| -tid *tid*             | デバッグセッションの開始時に再開されるスレッドのスレッド ID を指定します。                                                                      |
| -psn *ServiceName*     | デバッグするプロセスに含まれるサービスの名前を指定します。 これは、既に実行中のプロセスをデバッグするために使用されます。                           |
| -pn *ProcessName*      | デバッグするプロセスの名前を指定します。                                                                                                             |
| -z*ダンプファイル*          | デバッグするクラッシュダンプファイルの名前を指定します。 パスとファイル名にスペースが含まれている場合、引用符で囲む必要があります。                       |
| -debugArch x86-または-amd64    | 自動検出動作をオーバーライドし、デバッガーのターゲットビットを設定します。                                                                           |
| -loadSession           | 保存されているセッション構成ファイルを読み込みます。                                                                                                                      |
| -setupFirewallRules    | KDNET を使用したカーネルデバッグを許可するために、ローカルシステムで必要なファイアウォール規則を構成します。                                                                         |
| -openPrivateDumpByHandle*ハンドル* | *Microsoft 内部でのみ使用*します。 デバッグするクラッシュダンプファイルのハンドルを指定します。                                                                 |
| -benchmarkStartup      | *Microsoft 内部でのみ使用*します。 スタートアップベンチマークを実行し、結果をファイルに追加します。                                                                                                    |

**シンボルのオプション**

オプション | 説明
|------ | -----------|
-y*シンボルパス* | 使用するシンボルパスを指定します。 複数のパスはセミコロン ( **;** ) で区切ります。 パスにスペースが含まれている場合は、引用符で囲む必要があります。 詳細およびこのパスを変更するその他の方法については、「[シンボルパス](symbol-path.md)」を参照してください。
-n              | ノイズがあるシンボルの読み込み。 シンボルハンドラーからの詳細出力を有効にします。
-i *ImagePath*  | 使用するイメージの検索パスを設定します。
-sdce           | シンボルの読み込み中に、デバッガーに ' ファイルアクセスエラー ' メッセージが表示されるようにします。
-ses            | デバッガーがすべてのシンボルファイルの厳密な評価を実行し、疑わしいシンボルをすべて無視します。
-sicv           | シンボルハンドラーが CV レコードを無視するようにします。
-sins           | デバッガーがシンボルパスと実行可能イメージパス環境変数を無視するようにします。
-snc            | デバッガーが変換をオフC++にします。
-snul           | 非修飾名に対するシンボルの自動読み込みを無効にします。
-sup            | シンボルを検索するたびにシンボルハンドラーがパブリックシンボルテーブルを検索するようにします。
-sflags         | すべてのシンボルハンドラーオプションを一度に設定します。

**ソースパスのオプション**

 オプション   | 説明
|-------- | -----------|
-srcpath  | デバッグサーバーで使用するソースパスを指定します。
-lsrcpath | ローカルクライアントで使用するソースパスを指定します。

ローカルデバッガーセッションの場合、srcpath と lsrcpath は実質的に同じです ("サーバー" はローカルセッションです)。 リモートデバッグの場合、situtations には、これらを異なる値に設定する必要がある場合があります。 リモートデバッグの詳細については、「[リモートデバッグ](remote-debugging.md)」を参照してください。

**例外処理**

オプション | 説明
|------ | -----------|
-x   |   アクセス違反例外に対してのみ、2回目の処理を有効にします。
-xe*例外*  |   指定された例外に対して初回例外処理を有効にします。
-xd*例外* |   指定された例外に対して、2回目の例外処理を有効にします。
-xn*例外* |   指定された例外については、最初の処理と2回目の処理を無効にし、コンソールでのみメッセージを表示します。
-xi *Exception* |   指定された例外を完全に無視し、最初の処理と2回目の処理を無効にして、コンソールには何も outputing しません。

指定できる例外の一覧については、「[イベント定義と既定値](https://docs.microsoft.com/windows-hardware/drivers/debugger/controlling-exceptions-and-events#event-definitions-and-defaults)」を参照してください。

**事後分析**

オプション  | 説明
|------ | -----------|
-I     | システムの既定の事後分析デバッガーとして、WinDbg Preview を設定します。
-が    | WinDbg Preview をシステムの既定の事後分析デバッガーとして設定します。エラーのみが報告されます。
