# <a name="function-tables-for-miniport-drivers"></a>ミニポート ドライバーの関数テーブル


## <span id="function_tables_for_miniport_drivers"></span><span id="FUNCTION_TABLES_FOR_MINIPORT_DRIVERS"></span>


ジェネリックのミニポート ドライバーの上端のインターフェイス (を参照してください[WDM オーディオ用語](wdm-audio-terminology.md)) 関数のテーブルで構成されます。 一部の非オーディオ ミニポート ドライバーでは、ミニポート ドライバー、ミニポート ドライバーが必要になるコンテキスト構造体のサイズのポート ドライバーに通知する時に、登録時にポート ドライバーには、関数テーブルを指定します。 ポート ドライバーでは、プライベートの場所に関数のテーブルをコピーして context 構造体を割り当てし、コンテキスト構造体へのポインターを渡して、関数のテーブルの初期化関数を呼び出します。

同様に、オーディオのミニポート ドライバーを使用して、関数のテーブルが静的に割り当てられているポート ドライバーでコピーする必要はありません。 ポート ドライバーは、指定したプールからのコンテキスト (「オブジェクト」) のメモリを取得し、コンテキストに関数のテーブルにポインターをインストールします。 関数のテーブルのポインターは、コンテキストの最初のフィールドでは常にであるため、ポート ドライバーはコンテキスト ポインターのみを必要があり、コンテキストで関数テーブルにアクセスできます。
