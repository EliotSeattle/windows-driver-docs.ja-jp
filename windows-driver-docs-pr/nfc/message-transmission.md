# <a name="message-transmission"></a>メッセージの転送


NFP テクノロジは、近接テクノロジで定義されている別のデバイスが近接、判断した場合は、これがデバイス間で公開されたメッセージを送信するトリガーとして機能します。

公開されたメッセージは、デバイスが近接内では、厳密に 1 回送信する必要があります。

現在公開されているすべてのメッセージは、デバイスが近接内では、たびに送信する必要があります。

デバイスが近接し、トリガーがアクティブにメッセージが発行される場合、そのメッセージが送信する必要があります。

近接デバイスへの発行の配信では、パブリケーションが、NFP プロバイダーによって発行されていないする必要があります。 パブリケーションをクライアントを非公開にするまでは、次の近接イベントをアクティブに保ちます。

NFP デバイス ドライバー インターフェイスでは、公開されたメッセージが発行した順序で送信されることは必要ありません。 NFP デバイス ドライバー インターフェイスでは、1 つのブロックとして公開されたすべてのメッセージを転送することは必要ありません。 NFC は具体的にはこれを回避する相互運用機能の要件の詳細な: 1 つのブロックが許可されていません、複数のメッセージを送信します。 ただし、完全に、エラーなしに、個々 の公開されたメッセージが受信されることが必要ですまたはサブスクライブしているクライアントに渡される必要がありますされません。

受信側のデバイス上のすべてのアプリがメッセージにサブスクライブしている場合は、クライアントを通知するために、NFP デバイス ドライバー インターフェイスで定義されているメカニズムはありません。 またにサブスクライブされていないメッセージが受信されたことをクライアントに通知するためのメカニズムはありません。
