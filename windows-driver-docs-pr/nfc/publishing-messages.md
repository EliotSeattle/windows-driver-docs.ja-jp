# <a name="publishing-messages"></a>メッセージの発行


クライアントは、特定の種類でメッセージを発行できますが、サポート技術情報のメッセージを受信するユーザーまたは別のアプリまたはデバイスでメッセージを受け取ったときに制限されます。 これにより、簡単に「通知」のメッセージ送信のタイミングを管理することがなく目的のユーザーに情報をクライアントができます。

Windows 近接 API のクライアントは、メッセージを発行する要求できます。 パブリケーションの有効期間は、クライアントにバインドされます。 クライアントが実行されている限り、パブリケーションを維持できます。 いつでもでも、クライアントは、パブリケーションを削除できます (によって非公開) です。 クライアントの実行が停止している場合、パブリケーションが削除されます。

メッセージがパブリッシュできないときにデバイスが近接、将来に送信する必要がありますできなくします。

NFP モデルの目的は、通常は 512 バイト未満の短い (および迅速に送信されるため) のメッセージをサポートします。 NFP プロバイダーは、10 キロバイト (KB) を含むメッセージをサポートするためにのみ必要です。 クライアントは、すべてのプロバイダーで移植可能なコードのサイズの大きいメッセージを送信する必要があります、手動でメッセージのチャンクまたは帯域外のメッセージを送信します。

## <a name="message-transmitted-callback"></a>メッセージに送信されるコールバック


メッセージの送信されるコールバックは、発行されるメッセージが別のデバイスに送信されたことを認識するクライアントを許可します。 転送済みであるとすぐにメッセージを非公開にするアプリを有効にするメッセージの送信されたコールバックの目標の 1 つです。

モデルは、実際には、メッセージの処理、受信側の知識は提供されません。 これを実現する唯一の方法では、受信側メッセージをクライアントに発行します。
