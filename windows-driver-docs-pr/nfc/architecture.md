# <a name="nfc-class-extension-architecture"></a>NFC クラス拡張のアーキテクチャ


NFC ドライバーはクラス拡張として実装され、基になるトランスポートドライバーはクライアントドライバーとして実装されます。 モノリシックドライバーよりも主な利点は、将来的には、クライアントトランスポートドライバーを交換して追加のトランスポートをサポートすること、または、まだ NFC によって標準化されていない機能のチップの製造元の特定のニーズをサポートすることです。フォーラム.

クラス拡張のサポートは、UMDF 2.0 に含まれています。 NFC スタックは、カーネルモードで使用可能なコアシステムコンポーネントと、424Kbps に制限されているテクノロジによって暗黙的に示されるパフォーマンス要件に依存していないため、このドライバーがカーネルモードで機能する理由はありません。

| ファイル          | 説明                                                                                                                                                                                                                                                                                                                                                                                       |
|---------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| NfcCx .dll     | この DLL には、NFC クラスドライバーの実装が含まれています。 これは、UMDF に依存し、コンポーネントマニフェストを使用してインストールされます。 DLL は、コアシステムで使用できるものを上回る依存関係を持たない、システムに準拠したコアバイナリです。 DLL は、クライアントドライバーが NfcCxStub ライブラリを介して間接的にリンクしています。このライブラリを使用すると、クライアントドライバーによる読み込みと初期化の開始を行うことができます。 |
| NfcCxStub .lib | このファイルは、NfcCx .lib に直接リンクせずに、クライアントドライバーが NfcCx への読み込み時のリンクを実行できるスタブライブラリです。                                                                                                                                                                                                                                                     |

 

NFC クラス拡張ドライバーを更新 OS コンテキストで実行することは想定されていません。 ただし、ドライバーは、行末のテストを実行するために、Microsoft の製造 OS (MMOS) で実行されることが予想されます。 チップセット製造元によって提供される NFC クライアントドライバーでは、製造および最終ラインテストのために追加の DDI サポートを実装できますが、これはこのドキュメントの範囲外です。
