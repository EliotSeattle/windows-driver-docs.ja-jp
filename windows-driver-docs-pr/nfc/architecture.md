# <a name="nfc-class-extension-architecture"></a>NFC クラスの拡張機能アーキテクチャ


NFC ドライバーは、クラスの拡張機能として実装され、基になるトランスポート ドライバーは、クライアント ドライバーとして実装されます。 モノリシック ドライバー経由での主な利点は、クライアント トランスポート ドライバーできます置き換えられること、将来追加のトランスポートをサポートするために、または、NFC で標準化されていない機能の半導体メーカーの特定のニーズをサポートするにはフォーラム。

UMDF 2.0 には、クラス拡張のサポートが含まれます。 NFC スタックには、カーネル モード ドライバーとパフォーマンスの要件は、424 Kbps に制限するテクノロジによって暗黙的に指定で使用できるコアのシステム コンポーネントの依存関係があるない、ために、カーネル モードで機能するには、このドライバーの理由はありません。

| ファイル          | 説明                                                                                                                                                                                                                                                                                                                                                                                       |
|---------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| NfcCx.dll     | この DLL には、NFC クラス ドライバーの実装が含まれています。 UMDF に依存し、コンポーネント マニフェストを使用してインストールします。 DLL は、コア システム準拠上記すべての依存関係を持たないバイナリは、コア システムで使用できます。 DLL に直接リンクしないクライアント ドライバーを読み込むし、その初期化を開始できるようにする NfcCxStub ライブラリを使用したクライアント ドライバーによって。 |
| NfcCxStub.lib | このファイルは、読み込み時の直接リンク NfcCx.lib せず NfcCx.dll にリンクを実行するクライアント ドライバーをできるようにするスタブ ライブラリです。                                                                                                                                                                                                                                                     |

 

NFC クラスの拡張機能ドライバーが OS の更新プログラムのコンテキストで実行する必要はありません。 ただし、ドライバーは、実行で Microsoft 製造 OS (MMOS) 行の終わりのテストを実行すると想定されます。 チップセット製造元から提供された NFC クライアント ドライバーが製造および行の終わりのテスト目的で、追加 DDI サポートを実装できますが、このドキュメントのスコープ外であります。
