# <a name="identifying-backup-files-for-deleted-boot-entries"></a>削除されたブート エントリに対するバックアップ ファイルの識別

通常、ブート エントリが誤って削除されたときに、ブート エントリのバックアップ ファイルを検索する必要があります。

NVRAM からブート エントリが削除されて、オペレーティング システムがまだインストールされている場合は、ブート ローダーのファイルとインストールのブート エントリのバックアップ ファイル引き続き、ディスク上で EFI パーティション上のインストールのディレクトリ。

削除されたエントリのブート エントリのバックアップ ファイルを検索する、EFI シェルを起動し、コマンドを使用して、ブート エントリのバックアップ ファイルの EFI パーティション再帰的に検索**dir ブート\*/s**します。 結果からはディレクトリの NVRAM 内の既存のブート エントリに関連付けられているブート エントリのバックアップ ファイルを除外します。 既存のブート エントリのディレクトリを表示する使用、 **nvrboot d** (表示) コマンド。

複数のブートがある場合*xxxx*既存のブート エントリに関連付けられていないファイル Nvrboot を使用して、バックアップ ファイルからエントリをインポートして、望ましくないブート エントリを削除します。
