# <a name="static-and-dynamic-verification-tools"></a>静的および動的な検証ツール


検証ツールには、次の2つの基本的な種類があります。

-   **静的検証ツール**は、ドライバーを実行せずにドライバーコードを調べます。 これらのツールはコードを実行するテストに依存しないため、非常に詳細な場合があります。 理論的には、静的な検証ツールは、実際には実行されないコードパスを含むすべてのドライバーコードを調べることができます。 ただし、ドライバーは実際には実行されていないため、偽陽性の結果が生成される可能性があります。 つまり、実際には発生しない可能性のあるコードパスでエラーが報告される可能性があります。

-   **動的検証ツール**は、ドライバーの実行中にドライバーコードを調査します。通常は、一般的に使用される[ドライバーサポートルーチン](https://docs.microsoft.com/windows-hardware/drivers/ddi/index)への呼び出しをインターセプトし、同じルーチンのエラーチェックバージョンの呼び出しを置き換えます。 動的ツールが検証を実行している間、ドライバーは実際に実行されているので、偽陽性の結果はめったに発生しません。 ただし、動的ツールはドライバーの監視中に発生するアクションのみを検出するため、ドライバーのテストカバレッジが十分でない場合、ツールは特定のドライバーの欠陥を見逃す可能性があります。 同時に、実行時に使用可能な情報 (ソースコードから静的に抽出するのが困難な情報など) を使用することにより、動的検証ツールは、静的分析では検出しにくい特定のクラスのドライバーエラーを検出できます。用.

ベストプラクティスとして、静的検証ツールと動的検証ツールを組み合わせて使用することをお勧めします。 静的ツールを使用すると、実際には実行が困難なコードパスを確認できます。動的ツールでは、ドライバーで発生している重大なエラーが検出されます。
