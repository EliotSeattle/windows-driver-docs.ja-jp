# <a name="power-states"></a>電源の状態





電源の状態は、電力消費量のレベルを示します: とアクティビティのコンピューティングの範囲ではそのため、システムまたは 1 つのデバイス。 電源マネージャーは、全体として、システムの電源状態を設定します。 デバイス ドライバーは、個々 のデバイスの電源状態を設定します。

ACPI の仕様には、2 つの個別の電源状態のセットが定義されています:*システム電源の状態*と*デバイスの電源状態*します。 各電源状態では、一意の名前を持ちます。

[システムの電源状態](system-power-states.md)という名前が S*x*ここで、 *x*は 0 ~ 5 で状態の数です。 [デバイスの電源状態](device-power-states.md)D の名前は*x*ここで、 *x*は 0 ~ 3 の範囲の状態の数です。 状態の数が電力消費量に関連する反比例: 以上の番号付きの状態を使用して、電力を節約します。 状態の S0、D0 は highest-powered、最も機能で、完全に状態。 状態 S5 と D3 は最下位の電源状態であり、最長のウェイク アップ待機時間を持っています。

これらの電源を明確に定義された状態では、一貫して予想を連携するさまざまな製造元から多くのデバイスを許可します。 たとえば、S3 の状態で、システムを設定すると、電源マネージャーは、対応するデバイスの電源状態で自分のデバイスを配置するだけでなく、予測可能な形で稼働状態に戻すことも、電源管理をサポートするドライバーに利用できます。
