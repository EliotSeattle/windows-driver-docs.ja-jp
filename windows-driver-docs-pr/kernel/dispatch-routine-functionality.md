# <a name="dispatch-routine-functionality"></a>ディスパッチ ルーチンの機能





特定のディスパッチ ルーチンの必要な機能は、基になる物理デバイスの種類と、ドライバーのチェーン内の個々 のドライバーでの位置で、処理、I/O 関数のコードによって異なります。

ディスパッチ ルーチンのほとんどは、次のような受信 I/O 要求パケット (Irp) を処理します。

1. IRP を実行し、有効性、存在する場合、パラメーターを確認してください。 何かを判断するまでに、ドライバーの I/O スタックの場所を確認します。

   ドライバーで決定を行うと、パラメーターを確認するには、I/O スタックの場所を確認する必要があるかどうかによって異なります、指定された**IRP\_MJ\_**<em>XXX</em>、およびそのドライバーを設定するかどうかをそれぞれ別のディスパッチ ルーチンをサインアップ**IRP\_MJ\_**<em>XXX</em>ドライバーを処理します。

2. 要求を満たすし、可能であれば、IRP の完了それ以外の場合、転送することより低いレベルのドライバーまたはその他のデバイス ドライバーのルーチンをさらに処理します。

   はなく存在する場合、パラメーターの有効性に依存、ドライバーをさらに処理の IRP で渡す必要があるかどうか、 **IRP\_MJ\_**<em>XXX</em>とドライバーのレベルの場合それを階層型ドライバーのチェーンにします。

Irp の詳細については、次を参照してください。 [Irp の処理](handling-irps.md)します。
