# <a name="windows-kernel-mode-io-manager"></a>Windows カーネルモード I/O マネージャー


コンピューターは、外部との入出力 (i/o) を提供するさまざまなデバイスで構成されています。 一般的なデバイスは、キーボード、マウス、オーディオコントローラー、ビデオコントローラー、ディスクドライブ、ネットワークポートなどです。 デバイスドライバーは、デバイスとオペレーティングシステムの間のソフトウェア接続を提供します。 このため、デバイスドライバーライターにとって i/o は非常に重要です。

Windows カーネルモード i/o マネージャーは、アプリケーションと、デバイスドライバーによって提供されるインターフェイスとの間の通信を管理します。 デバイスはオペレーティングシステムと一致しない速度で動作するため、オペレーティングシステムとデバイスドライバーの間の通信は、主に i/o 要求パケット (Irp) を介して行われます。 これらのパケットは、ネットワークパケットや Windows メッセージパケットに似ています。 これらは、オペレーティングシステムから特定のドライバー、およびあるドライバーから別のドライバーに渡されます。

Windows i/o システムは、stack と呼ばれるレイヤー化されたドライバーモデルを提供します。 通常、Irp は、通信を容易にするために、同じスタック内のあるドライバーから別のドライバーに送られます。 たとえば、ジョイスティックドライバーは USB ハブと通信する必要があり、usb ホストコントローラーと通信する必要があります。その後、PCI バスを介してコンピューターハードウェアの残りの部分に通信する必要があります。 スタックは、ジョイスティックドライバー、USB ハブ、USB ホストコントローラー、および PCI バスで構成されます。 この通信は、スタック内の各ドライバーが Irp を送受信することによって調整されます。

ドライバーは、スタック全体が効率的に動作するために、Irp をタイムリーに送受信する必要があるため、十分な負荷をかけることはできません。 ドライバーがスタックの一部で、情報を適切に受信、処理、および渡すことができない場合は、ドライバーによってシステムがクラッシュする可能性があります。

Irp の詳細については、「 [irp の処理](handling-irps.md)」を参照してください。

ドライバースタックの詳細については、「[デバイスオブジェクトとデバイススタック](device-objects-and-device-stacks.md)」を参照してください。

I/o 管理に関連するプログラミングについては、「 [I/o マネージャーのプログラミング手法](i-o-programming-techniques.md)」を参照してください。

I/o マネージャーに直接インターフェイスを提供するルーチンには、通常、"**Io**" という文字が付いています。たとえば、 **IoCreateDevice**のようになります。 I/o マネージャールーチンの一覧については、「 [I/o マネージャールーチン](https://docs.microsoft.com/previous-versions/windows/hardware/drivers/ff551797(v=vs.85))」を参照してください。

IRP に関連するルーチンの一覧については、「 [irp](https://docs.microsoft.com/windows-hardware/drivers/ddi/index)」を参照してください。

I/o マネージャーには、プラグアンドプレイ manager と電源マネージャーの2つのサブコンポーネントがあります。 プラグアンドプレイと電源管理のテクノロジの i/o 機能を管理します。 プラグアンドプレイ管理の詳細については、「 [Windows カーネルモードのプラグアンドプレイマネージャー](windows-kernel-mode-plug-and-play-manager.md) 」を参照してください。電源管理の詳細については、「 [windows カーネルモードの電源マネージャー](windows-kernel-mode-power-manager.md)」を参照してください。
