# <a name="supporting-multifunction-devices-on-other-buses"></a>その他のバス上に多機能デバイスのサポート





PnP ISA、USB、または IEEE 1394 のバスの多機能デバイスの場合は、親のバス ドライバーは、デバイスは、バスの標準に準拠している場合に個々 の関数を列挙します。

このようなデバイスの場合は、親のバス ドライバーは、1 つのバスの場所に存在する 1 つ以上のデバイスがあるという事実を管理します。 システムの残りの部分には、個々 の関数は、独立したデバイスと同様に動作します。

この種類の多機能デバイスのベンダーでは、次の操作を行う必要があります。

-   デバイスが、デバイスが存在するバスの仕様に準拠していることを確認します。

-   デバイスの各関数の関数の PnP ドライバーを提供します。

    システム提供のバス ドライバーでは、多機能のセマンティクスを処理するため、関数のドライバーは、関数が個々 のデバイスとしてパッケージ化するときに使用される同じドライバーを指定できます。

-   各関数の場合、デバイスの INF ファイルを提供します。

    INF ファイルには、関数が個々 のデバイスとしてパッケージ化するときに使用される同じファイルを指定できます。 INF ファイルでは、多機能の特殊なセマンティクスは必要ありません。
