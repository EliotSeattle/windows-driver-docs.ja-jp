# <a name="mb-driver-model-versioning"></a>MB ドライバー モデルのバージョン管理


MB ドライバーモデルのバージョン管理は、ドライバーモデルのバージョンと個々の OID データ構造のリビジョンを持つことで実現されます。 これは、NDIS 6.x で使用されるバージョン管理パラダイムと一致します。

ドライバーモデルのバージョンは、MB サービスと MB ミニポートドライバーの間のインターフェイスの進化を定義します。 個々の OID リビジョンでは、異なる MB のドライバーモデルバージョンで Oid に加えられた変更を追跡します。 つまり、ドライバーモデルのバージョンは、特定のリビジョン番号によって識別されるデータ構造を持つ Oid のセットを定義します。

*NDIS 仕様*と一貫して、MB ドライバーモデルの進化は*加法*です。 つまり、新しい Oid と新しいメンバーは、既存の OID データ構造にのみ追加できます。 これにより、MB サービスでミニポートドライバーの旧バージョンとの互換性がサポートされるようになります。

**重要**  非常にまれな状況でのみ、既存の oid が非推奨になるか、または既存の oid データ構造のメンバーが次のバージョンで使用されなくなります。 そのような場合、これらの変更と下位互換性への影響は、新しいバージョンの MB ドライバーモデル仕様に関する以降のドキュメントで明確に記載されているものとします。

 

このドキュメントでは、MB ドライバーモデルの Windows 8 リリースについて説明します。 ドライバーモデルのバージョンは、バージョン2.0 にインクリメントされました。 一部の OID のリビジョンは、リビジョン2に更新されていますが、リビジョン番号1と続きます。 各 Oid で使用するリビジョンの詳細については、「 [MB データモデル](mb-data-model.md)」を参照してください。

このドキュメントでは、MB ドライバーモデルの最初のリリースについて説明します。そのため、ドライバーモデルのバージョンと個々の OID のリビジョンの両方がリビジョン番号1で始まります。

ドライバーモデルが次のバージョンに移動すると、そのバージョン番号が1ずつ増加します。 ドライバーモデルに追加された新しい Oid は、リビジョン1から開始されます。データ構造が変更された既存の Oid では、対応するリビジョンが1ずつ増加し、変更されていない既存の Oid はそれぞれのリビジョン番号を保持します。

ドライバーモデルのバージョンは、 [OID\_WWAN\_ドライバー\_cap](https://docs.microsoft.com/windows-hardware/drivers/network/oid-wwan-driver-caps)によって伝達されます。 Mb サービスは、 [Mb ミニポートドライバーの初期化](mb-miniport-driver-initialization.md)中に、OID\_WWAN\_ドライバー\_cap クエリ要求をミニポートドライバーに送信します。 個々の OID のリビジョンは、個々の OID のデータ構造の一部として含まれる[**NDIS\_オブジェクト\_ヘッダー**](https://docs.microsoft.com/windows-hardware/drivers/ddi/ntddndis/ns-ntddndis-_ndis_object_header)構造の**リビジョン**メンバーによって記述されます。
