# <a name="security-checklist-for-ndis-drivers"></a>NDIS ドライバーのセキュリティ チェックリスト





ドライバーが適切なセキュリティ プラクティスに従うことを確認するには、次の操作を行います。

-   可能な場合は、何らかの理由でパケットのペイロードの情報を解析するコードを回避します。 ドライバーのパケットが処理ディスパッチ ルーチンからのオフロード検証の処理では特に、このようなコードを削除することをお勧めします。

-   ドライバーの送信の確認コード パスが表示され、何らかの理由でパケットのペイロードの情報を解析するすべてのコードを慎重に確認します。

-   十分にセキュリティ ホールのドライバーのコードを確認し、ドライバーをリリースする前に、ドライバーをテストします。 すべてのエラー パスだけではなく、通常のコード パスを確認することを確認します。

-   ドライバーが正しくないパケット情報を抵抗できるかどうかを確認するランダムなパケット生成テストを実行します。 今後、このようなテストは、デバイスのロゴ認証を必須になります。
