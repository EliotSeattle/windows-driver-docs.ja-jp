# <a name="added-costs-for-testing-vulnerable-ndis-drivers"></a>脆弱な NDIS ドライバーをテストするための追加のコスト





ドライバーのパケットが処理ディスパッチ ルーチンからのオフロード検証の処理では特に、パケットのペイロードを解析するコードを削除することをお勧めします。 このようなコードにある信頼度は、するには、広範囲にテストして、ドライバーをすべての潜在的なエラー状態が安全かつ正常に処理されるかどうかを確認する必要があります。 この種類の意味をテストには、テスト コストが増加します。
