# <a name="wdi-datapath-architecture"></a>WDI データパス アーキテクチャ


既存の WLAN のターゲット デバイスを相互運用するには、ドライバー インターフェイスの現在のバージョンは、送信/受信用ホスト コント ローラー インターフェイス (HCI) を指定しません。 代わりに、WDI とターゲット アダプテーション層 (話して) 間の要求/indication ソフトウェア インターフェイスを指定します。

WDI コンポーネントは、NDIS データパス インターフェイスおよびホストで実行されるターゲットに依存しない TX/RX 関数を実装します。 個々 の仮想 WLAN デバイス (ポート) の状態およびピア固有の状態も保持されます。

コント ローラーとバスのインターフェイスの機能だけでなく、実装、ホスト コント ローラー インターフェイスに依存する TX/RX WLAN 関数、話してを提供します。

![wdi データパス ブロック図](images/wdi-datapath-block-diagram.png)

TX/RX 関数だけでなく、話しては制御とデータ パスを使ってターゲット インターフェイス レイヤー (TIL) を提供します。 TIL の責任は、次の表に一覧表示されます。

| 関数が                            | 説明                                                                                                                                                                      |
|-----------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ホスト ターゲットの通信の管理 | 以下に例を示します。割り当てし、管理の制御とデータのパスに必要な DMA チャネル。                                                                                               |
| バスの適応/抽象化              | ホスト/ターゲットの標準的な通信を異なるバス タイプ、ソフトウェア/ハードウェア バス エンドポイント、およびバス DMA エンジン間通信の違いを抽象化する API を提供します。 |
