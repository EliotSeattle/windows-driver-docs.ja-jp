# <a name="packet-inspection-points"></a>パケット検査ポイント


次の順序で WFP レイヤーを受信側のコンピューター (ローカル ホスト トラフィック) の走査に割り当てられているアドレス宛ての受信パケットは:

<a href="" id="ip-packet--network-layer-"></a>**IP パケット (ネットワーク層)**  
IP パケットのフラグメントを含むすべての IP パケットは、このレイヤーで検査できます。 ただし、パケットが IPsec で保護された、高度なコンテンツの検査と変更は実行できませんこのレイヤー パケットがないため、まだ認証または暗号化を解除します。

<a href="" id="transport-layer"></a>**トランスポート層**  
すべてのスタンドアロンまたは完全に再構築されたパケットは、このレイヤーで検査に使用されます。 パケットの IPsec で保護された認証または暗号化解除されました。

<a href="" id="application-layer-enforcement--ale--receive-or-accept"></a>**アプリケーション レイヤーの強制 (ALE) が表示されるか、そのまま使用**  
このレイヤーでは、ローカル エンドポイントに到着する最初のパケットが示されます。 たとえば、到着の TCP 同期 (SYN) セグメントまたは UDP フローに関連付けられている最初の UDP メッセージが示されます。 ファイアウォール ポリシーの変更後に、接続を再承認を必要とされるパケットは、このレイヤーでも示されます、ALE の再認証フラグが設定されます。

<a href="" id="datagram-data-or-stream"></a>**データグラム データまたは Stream**  
データグラム データ層には、UDP メッセージおよび非 ICMP エラー メッセージが示されます。 TCP データ フロー (データ ストリーム) ストリーム レイヤーで検査に使用されます。

WFP の次のレイヤーを送信元のコンピューター (ソースとローカル ホスト トラフィック) の走査に割り当てられているアドレスから発信された送信パケット:

<a href="" id="ale-connect"></a>**ALE 接続します。**  
TCP 接続 (SYN セグメントを生成する前に行われた) 要求とリモート エンドポイントに送信される最初の UDP メッセージは、このレイヤーで示されています。

<a href="" id="datagram-data-or-stream"></a>**データグラム データまたは Stream**  

<a href="" id="transport-and-icmp-error"></a>**トランスポートと ICMP のエラー**  
このレイヤーには、TCP 接続要求 (SYN セグメントを生成) する前に、リモート エンドポイントに送信される最初の UDP メッセージが示されます。

<a href="" id="ip-packet"></a>**IP パケット**  
IP パケットのフラグメントが反映されていません。送信 IP フラグメントの検査では、現在使用できません。

IP パケットまたはフラグメントのものでないかの宛てではない、ローカル コンピューターに割り当てられているアドレスが転送層で検査します。 たとえば、ローカル クライアントに送信されるパケットは、非ローカルの宛先アドレスが変更され、受信パスに挿入されますが場合、転送のレイヤーに挿入されます。 同様に、ローカル ソース アドレスから送信されたパケットが非ローカルのソース アドレスが変更された場合に配信されます転送層に送信パスに挿入されてその後。
