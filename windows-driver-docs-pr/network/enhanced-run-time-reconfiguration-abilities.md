# <a name="enhanced-run-time-reconfiguration-abilities"></a>実行時再構成機能の拡張





NDIS 6.0 には、一時停止してスタックを破棄して新しいゲートウェイを作成することがなくドライバー スタックを再開する機能が導入されました。 NDIS 6.0 とそれ以降のドライバーのすべてが一時停止をサポートし、サービスを再起動する必要があります。

再構成が行われる前に、既知の状態ですべてのドライバーを配置することで同期の問題を排除するスタックを一時停止します。 一時停止する機能も、NDIS、ドライバーの特性を照会し、スタックの他の特性を再構成できます。

NDIS は、ドライバー スタックを一時停止の追加や削除など、フィルター ドライバーまたはバインド プロトコル ドライバーをバインド解除のプラグ アンド プレイ操作を実行する前に、データ フローを一時的に停止することができます。 NDIS は、再構成が行われた後に、スタックを再起動します。

ミニポートおよびフィルター ドライバーは、一時停止を処理し、関数のインターフェイスを介してサービスを再起動します。 プロトコル ドライバーでは、一時停止を処理し、プラグ アンド プレイのイベント通知によってサービスを再起動します。

詳細については一時停止し、再起動操作を参照してください。[ドライバー スタック管理](driver-stack-management.md)します。
