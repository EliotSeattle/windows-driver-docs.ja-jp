# <a name="modifying-a-running-driver-stack"></a>実行中のドライバー スタックを変更します。





NDIS は、挿入、削除、またはフィルター モジュールの再構成などの操作用のドライバー スタックを変更します。 NDIS では、アクティブ化したり、フィルター モジュールでバイパス モードを非アクティブ化することができます。 フィルター ドライバーでバイパス モードの詳細については、次を参照してください。[データ バイパス モード](data-bypass-mode.md)します。

**注**  場合ドライバーのエントリ ポイントの変更をフィルター処理 (は、バイパス モードのため)、NDIS を一時停止し、ドライバー スタックを再起動します。 一時停止と再起動が発生するネットワーク パケットをドロップできるは、送信パスまたはパスを受信できませんでした。 信頼性の高いトランスポート メカニズムを提供するネットワーク プロトコルがネットワークの場合は、損失パケットを I/O 操作を再試行可能性がありますが、信頼性を保証しないその他のプロトコルは、操作を再試行しないでください。

 

NDIS は実行中のドライバー スタックとは、次のように変更します。

1.  NDIS は、ドライバー スタックを一時停止します。

    詳細については、次を参照してください。[ドライバー スタックを一時停止](pausing-a-driver-stack.md)します。

2.  NDIS は、スタックを変更します。

    たとえば、フィルター モジュールを追加するに NDIS スタックに新しいフィルター モジュールを挿入する場所を決定しますと作成、挿入、およびフィルター モジュールをアタッチします。

3.  フィルター モジュールが挿入または削除、ドライバー スタックの特性を変更する可能性があります。 この場合は、NDIS は、この変更のドライバーに通知するには、プロトコル バインドとフィルター モジュールではドライバー スタックのすべてにプラグ アンド プレイのイベント通知を送信します。

4.  NDIS は、ドライバー スタックを再起動します。

    詳細については、次を参照してください。[ドライバー スタックの再起動](restarting-a-driver-stack.md)します。
