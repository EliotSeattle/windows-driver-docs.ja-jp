# <a name="multiprocessor-support-in-network-drivers"></a>ネットワーク ドライバーのマルチプロセッサ サポート





移植可能なすべての Microsoft Windows バージョンのドライバーを作成するには、プロセッサを同時に実行する複数のコンピューターで安全に実行するコードを記述する必要があります。 ネットワーク ドライバーは、マルチプロセッサ セーフである必要があり、指定された NDIS ライブラリ関数を使用する必要があります。

ユニプロセッサ環境で場合でも、ネットワーク インターフェイス カード (NIC) またはその他のデバイスのパケットが到着したとき、またはタイマー割り込みが発生すると、実行の現在のストリームを中断する可能性があります、1 つのプロセッサは、一度に 1 台のコンピューターの命令を実行します。 通常、パケット キューなどのデータ構造を操作するときドライバー NIC での割り込みを無効にします、操作を行い、割り込みを再び有効にし。 同時に実行しますが、インターリーブのタイム スライスで実際に実行する単一プロセッサ環境で多数のスレッドが表示されます。

マルチプロセッサ環境では、プロセッサは同時にコンピューターのいくつかの手順を実行します。 ドライバーには、ときにドライバー機能の 1 つは同じ共通のデータ構造を操作したり、別のプロセッサで別のドライバー関数は、同時に共有データを変更しようとはしませんを同期する必要があります。 すべてのドライバー コードは、対称マルチプロセッサ (SMP) コンピューターで再入可能です。 このリソースの保護の問題を取り除くには、Windows デバイス ドライバーは、スピン ロックを使用します。 詳細については、次を参照してください。[同期とネットワーク ドライバーに通知](synchronization-and-notification-in-network-drivers.md)します。
