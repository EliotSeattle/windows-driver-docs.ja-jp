# <a name="types-of-physical-network-adapter-configurations"></a>物理ネットワーク アダプターの構成の種類


HYPER-V 拡張可能スイッチのアーキテクチャでは、基になる物理メディアにアクセスするため、1 つの外部ネットワーク アダプターへの接続をサポートしています。 外部ネットワーク アダプターは、次の物理ネットワーク アダプター構成のいずれかにバインドできます。

-   外部ネットワーク アダプターは、1 つの基になる物理ネットワーク アダプターにバインドできます。 この構成では、拡張可能スイッチの拡張機能に公開し、ホスト上の 1 つだけの基になるネットワーク アダプターを管理します。

    次の図は、外部ネットワーク アダプターが NDIS 6.40 (Windows Server 2012 R2) 以降は、1 つの物理ネットワーク アダプターにバインドされている拡張可能スイッチの構成を示します。

    ![ndis 6.40 およびそれ以降の 1 つの物理ネットワーク アダプターにバインドされている外部のネットワーク アダプターと vswitch の構成](images/vswitchsingle-ndis640.png)

    次の図は、外部ネットワーク アダプターが NDIS 6.30 (Windows Server 2012) の 1 つの物理ネットワーク アダプターにバインドされた拡張可能スイッチの構成を示します。

    ![ndis 6.30 の単一の物理ネットワーク アダプターにバインドされる外部ネットワーク アダプターと vswitch の構成](images/vswitchsingle.png)

-   外部ネットワーク アダプターは、負荷分散のフェールオーバー (LBFO) プロバイダーの仮想ミニポート端にバインドできます。 これは、NDIS マルチプレクサー (マルチプレクサー) ドライバー、ホスト上の 1 つまたは複数の物理ネットワーク チームにバインドされる上に重ねられるする NDIS フィルター ドライバーです。 この構成と呼ばれる、 *LBFO チーム*します。

    この構成で拡張可能スイッチ拡張機能は、ネットワーク アダプターとしてを基になる仮想ミニポート エッジのみに公開されます。 これにより、複数の物理ネットワーク アダプターにバインドして、LBFO ソリューションをサポートするプロバイダーです。 これらの物理ネットワーク アダプターは、拡張可能スイッチのドライバー スタックで実行されている転送拡張機能で管理されていません。

    次の図は、NDIS 6.40 (Windows Server 2012 R2) と後では、LBFO チームの構成の例を示します。

    ![ndis 6.40 の lbfo チームの構成を示すフローチャート](images/vswitchteam3-ndis640.png)

    次の図は、NDIS 6.30 (Windows Server 2012) の LBFO チームの構成の例を示します。

    ![ndis 6.30 の lbfo チームの構成を示すフローチャート](images/vswitchteam3.png)

    **注**  拡張可能スイッチ拡張機能には、基になる LBFO チームが外部ネットワーク アダプターにバインドされている単一の仮想ネットワーク アダプターとして表示されます。

     

-   外部ネットワーク アダプターは、NDIS MUX の中間ドライバーの仮想ミニポート端にバインドできます。 MUX driver は、ホスト上の 1 つまたは複数の物理ネットワーク チームにバインドされます。 この構成と呼ばれる、*拡張可能スイッチ チーム*します。

    この構成で拡張可能スイッチの拡張機能は、チーム内のすべてのネットワーク アダプターに公開されます。 これにより、拡張機能の構成と、チーム内の個々 のネットワーク アダプターの使用を管理できます。 たとえば、拡張機能では、個々 のアダプターに送信されるパケットを転送することによって、over、チーム分散フェールオーバー (LBFO) のソリューション ロードのサポートを提供できます。

    次の図は、NDIS 6.40 (Windows Server 2012 R2) と後で拡張可能スイッチ チームの例を示します。

    ![拡張可能なを示すフローチャートは、ndis 6.40 のチームを切り替える](images/vswitchteam-ndis640.png)

    次の図は、NDIS 6.30 (Windows Server 2012) の拡張可能スイッチ チームの例を示します。

    ![拡張可能なを示すフローチャートは、ndis 6.30 のチームを切り替える](images/vswitchteam.png)

MUX ドライバーの詳細については、次を参照してください。 [NDIS MUX 中間ドライバー](ndis-mux-intermediate-drivers.md)します。
