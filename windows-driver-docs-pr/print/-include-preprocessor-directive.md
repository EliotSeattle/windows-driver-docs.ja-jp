# <a name="include-preprocessor-directive"></a>\#プリプロセッサ ディレクティブを含める


```GDL
#Include: Quoted String
```

\#インクルード ディレクティブによって指定される GDL ソース ファイルが実行*引用符で囲まれた文字列*が読み込まれ、処理します。 インクルード ファイルが処理されるまで、現在 GDL ファイルの前処理が一時停止します。 インクルード ファイルの定義またはシンボルを undefining によりホスト GDL ファイルの残りの部分の前処理に影響を与えることができます。

引用符で囲まれた文字列の構文は、GDL によって定義されます。 その他のディレクティブの値とは異なり、引用符で囲まれた文字列値を 1 つ以上の行の間で拡張できます。 *文字列を引用符で囲まれた*が必要です。

\#含めるし、中かっこ (}) ではなく、改行によってすべてのディレクティブを終了する必要があります。

使用する場合 **\*Include**、古い GPD キーワードは、インクルード ファイルは、ホスト ファイルの後に前処理します。 この処理には、ホスト ファイルには、最初に前処理されるインクルード ファイルが必要な場合に問題が発生します。 このような潜在的な問題を避けるためには、常にプレフィックス、\#現在プリプロセッサ プレフィックスを持つインクルード ディレクティブ。

パーサーの現在の実装は、3 つのファイルの名前付け形式を使用します。 ファイル名のみ、完全修飾パス、および部分的に修飾パス。 部分的に修飾パスを使用する場合、パスの開始点は、現在の実行環境によって確立されます。 ファイル名を使用すると、専用の場合、2 つの開始点が試行されます。 ルート ソース ファイルが使用するパスとし、現在の実行環境を確立するパス。

注プリコンパイル済みファイルには、別のファイルが含まれている場合、プリコンパイル済みファイルと見なされることのインクルード ファイル、ルート ソース ファイルが関係しています。 インストールとセットアップ コードは、追加の制限を課すことがあります。
