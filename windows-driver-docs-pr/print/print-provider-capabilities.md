# <a name="print-provider-capabilities"></a>印刷プロバイダー機能





**警告**  以降 Windows 10 では、サード パーティの印刷プロバイダーをサポートする Api は非推奨とされます。 Microsoft には、サード パーティの印刷プロバイダーに他の投資はお勧めしません。 さらに、Windows 8 および v4 印刷ドライバー モデルが使用可能な新しい製品では、サード パーティの印刷プロバイダーが作成や v4 印刷ドライバーを使用するキューを管理します。

 

API の定義済みセットをサポートすることでは、関数、Microsoft Windows 2000 およびプリント プロバイダー後では、次の機能を指定できます。

-   印刷キューの管理

    追加、削除、開始、終了、列挙、および印刷キューのパラメーターを設定します。 また、印刷キューの状態の変更の通知を提供します。

-   プリンター ドライバーの管理

    追加、削除、列挙、およびプリンター ドライバー用のディレクトリを指定します。

-   印刷ジョブの作成

    開始日と開始日と終了ポートにプリンターの状態情報を読み取り、ジョブのデータ ストリームを書き込み、ドキュメントのページ、ドキュメントを終了します。

-   印刷ジョブのスケジューリング

    スケジュール設定、列挙、および印刷ジョブのパラメーターを設定します。

-   フォームの管理

    追加、削除、列挙、および印刷のフォームのパラメーターを設定します。

-   プリント プロセッサの管理

    追加、削除、列挙、用のディレクトリとプリント プロセッサによってサポートされるデータ型を指定します。

-   印刷管理の監視

    追加、削除、および列挙するモニターを印刷します。

-   ポートの管理

    追加、削除、構成、列挙、およびプリンター ポートのパラメーターを設定します。

-   レジストリの管理

    作成、削除、およびレジストリ キーと印刷のプロバイダーに関連付けられた値を列挙します。

-   その他の機能

    印刷のプロバイダーのシャット ダウン、メッセージ ボックスを表示すると、ポート モニター UI の Dll とポートの監視のサーバー Dll 間の通信パスを提供する、スプール ファイルがマップされている、メモリの読み取り。

これらの機能のセットとして実装[印刷プロバイダーによって定義されている関数](functions-defined-by-print-providers.md)します。
