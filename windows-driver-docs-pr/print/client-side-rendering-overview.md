# <a name="client-side-rendering-overview"></a>クライアント側レンダリングの概要

[ポイント アンド プリント](introduction-to-point-and-print.md)以前のバージョンの Windows オペレーティング システムのようにクライアント コンピューターにプリンター ドライバーを読み込みます。 クライアント側のレンダリングには、拡張メタファイル (EMF) 形式またはプリンター ドライバーを使用する XML Paper Specification (XPS) 形式の代わりに、プリンターを使用するページ記述言語 (PDL) に印刷ジョブを表示するために、プリンター ドライバーが原因です。 RAW 形式 PDL は、キュー、および印刷スプーラー サービスの新機能での印刷とプリント サーバーに送信されます。

プリント サーバーからクライアント コンピューターに印刷ジョブのレンダリングの処理負荷を移動するだけでなくクライアント側のレンダリングは、ユーザーにもデータのこれらの利点を提供します。

-   ドライバーの不一致の問題が除外されます。

    印刷ジョブがスプールのと同じコンピューターでは、EMF 形式のデータもレンダリングするため、不要になった問題がある場合は、クライアントがあるプリント サーバーのバージョンと互換性のないプリンター ドライバーのバージョン。

-   オフライン印刷がサポートされています。

    プリンターをホストするプリント サーバーに接続されていない場合でも、エンドユーザーはリモート プリンターに印刷ジョブをスプールようになりましたことができます。 クライアント側のレンダリング機能では、ユーザーはスプールし、印刷ジョブをローカルでの表示にします。 クライアント コンピューターが、印刷スプーラーへの接続を確立時にレンダリングされた印刷ジョブはプリント サーバーに印刷用に自動的に送信されます。
