# <a name="gdl-preprocessor-syntax"></a>GDL プリプロセッサ構文


GDL プリプロセッサ ディレクティブは、次の規則に従う必要があります。

-   すべてのプリプロセッサ ディレクティブは、個別の行を占有する必要があり、その行の唯一のステートメントがあります。 省略可能な空白のみは、プリプロセッサ ディレクティブを先頭に付けることができます。 ファイルが解析の 2 つ目の (メイン) フェーズに送信される前に、同じ行に、ディレクティブに続く余分な文字が削除されます。

-   すべてのディレクティブは、現在のプリプロセッサのプレフィックスで接頭辞する必要があります。 プリプロセッサのプレフィックスは、アスタリスクをパーサーが初期設定は (\*) またはシャープ記号 (\#)、任意の文字またはトークンを使用してプレフィックスを変更することができますが、  **\#SetPPPrefix**ディレクティブ。

-   プリプロセッサ ディレクティブとして認識されるように、プリプロセッサのプレフィックスの直後に、ディレクティブとディレクティブには、値が必要ですが場合、値は、コロン (:) で区切る必要があります。

-   ディレクティブの値は、任意の空白または改行文字で終了します。

**注**   GDL 構文は GPD 構文よりもより厳密でないです。 両方のパーサーを作成する場合は、GPD に必要なより厳密な構文に従ってください。
