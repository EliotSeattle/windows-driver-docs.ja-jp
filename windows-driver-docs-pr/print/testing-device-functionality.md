# <a name="testing-device-functionality"></a>デバイスの機能をテストします。


デバイスのインストールとプラグ アンド プレイの機能のすべてのテストを完了すると後、は、デバイスが想定どおりに機能を実行することを確認します。 できるだけ多くのアプリケーションおよびドキュメントの種類のテスト ページを印刷します。

次の追加の機能テストを実行することも必要があります。

1.  **ユーザーのアクセス:** さまざまなレベルのシステムへのアクセスを持つユーザーは、リモート デスクトップ接続アプリケーションを使用してリモート システムにログオンしているユーザーを含め、デバイスに印刷できることを確認します。

2.  **特別な機能:** デバイスをサポートする任意の特別な機能を確認します。 たとえば、プリンターでは、双方向 (bidi) 通信を可能である場合は、プリンターから双方向の要求がデバイスと双方向印刷から印刷ジョブの取り消しなど、想定どおりに動作することを確認します。

3.  **ジョブ制御:** 印刷ジョブを一時停止、キャンセル、再開、および再起動できますを確認します。 また、複数の印刷ジョブを開始、印刷、取り消されると、および再起動できますと同時に確認します。
