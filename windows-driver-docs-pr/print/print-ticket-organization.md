# <a name="print-ticket-organization"></a>印刷チケットの構成


PrintTicket ドキュメントは、ドキュメントのさまざまな部分に適用されるコマンドを含めることができます。 印刷チケットのドキュメントには、それぞれの特異性に従って付けられます。 順位は次のコンテンツのレベルのいずれかを含めることができます。

-   高レベルのジョブ (ジョブレベルの印刷チケット)

-   ジョブ (ドキュメント レベルの印刷チケット) 内のドキュメント

-   ドキュメント (ページ レベルの印刷チケット) 内のページ

ジョブ レベルの印刷チケットでは、ドキュメント レベルの印刷チケットと最後に、ページ レベルの印刷チケットで後に、最も具体的である最も一般的です。 これらのレベルに適用される印刷スキーマのフレームワークの要素には"Job"、"Document"または「ページ」プレフィックスそれに応じて。 印刷チケットの階層は、XPS ドキュメントの部品の階層に対応します。

印刷チケットの階層的な性質によりより高度な印刷チケットの対応する要素をオーバーライドする印刷チケットを下位レベルの要素です。 PrintTickets を使用する前に、PrintTicket の特定の文書パーツの効果的な印刷チケットの取得に適用されるドキュメント内のより高いレベルのオブジェクトと結合する必要があります。 このマージは、効果的な印刷チケットが処理する場合に、印刷ドライバーのように要求されるまでに実行されます。
