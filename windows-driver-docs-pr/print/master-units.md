# <a name="master-units"></a>マスター ユニット





ほとんどのプリンターでは、さまざまな水平方向および垂直方向の解像度でコマンドをサポートします。 たとえば、特定のプリンターの即時のライン フィードのコマンドを取得解像度は 1/288th インチ、同一のプリンターが 1 の垂直方向のグラフィックの解像度をサポート可能性があります/96 インチです。 同様に、このプリンターが 1/80th、1/160th、および 1 の水平方向の解像度も提供インチ 320th/。

Unidrv では、これらのさまざまな解像度を処理するために 1 つの座標システムを提供します。 この座標系においてユニットは、マスターのユニットと呼ばれます。 プリンターのマスターの単位は、(*x*、 *y*) の値のペアを*x*水平方向の単位をマスターと*y*垂直方向のマスターの単位です。

平面のマスターの単位を確認するのには、実際の解像度の分母の最小公倍数 (LCM) を計算します。 この例のプリンターでは、次の操作を行うは。

-   これは、320 80、160、および、320 の LCM を計算します。 したがって、水平方向のマスター単位には 1 インチの 320th/。

-   これは 576 288 と 96 の LCM を計算します。 したがって、垂直方向のマスター単位には 1 インチの 576th/。

**重要な**  マスター単位の値と、垂直および水平方向の解像度の両方、印刷ヘッドのピンの数の倍数である必要があります (つまり、 **PinsPerPhysPass**値)。 この条件が満たされない場合は、特定の用紙サイズに余分な空白行が生成可能なは。


一般に、位置と GPD ファイルのエントリで使用するサイズ値は、マスターの単位で指定する必要があります。 たとえば、12 インチで、この例のプリンターのカスタム ページの最大サイズが 9 インチを指定するに次のエントリ使用される場合、9 x 320 = 2880 と 6912 576 = 12 x:
