# <a name="docking-versus-laptop-slate-conversion"></a>ドッキングとノート PC スレートの変換


このトピックでは、ラップトップのスレートの変換とアクションのドッキングのインジケーターの違いについて説明します。

## <a name="span-idlaptopslateconversionspanspan-idlaptopslateconversionspanspan-idlaptopslateconversionspanlaptop-slate-conversion"></a><span id="Laptop_slate_conversion"></span><span id="laptop_slate_conversion"></span><span id="LAPTOP_SLATE_CONVERSION"></span>スレートのラップトップの変換


スレートのラップトップの変換は、同時に (物理的に接続されている) のポータブルのシステムにユーザーがキーボードの可用性を変更するユーザー操作です。 このようなアクションの例は次のとおりです。

-   移植可能なキーボード アクセサリをアタッチ (と、バッテリ パックなし)。
-   反転、回して、または完全に接続されているキーボードは必要があります。

## <a name="span-iddockingspanspan-iddockingspanspan-iddockingspandocking"></a><span id="Docking"></span><span id="docking"></span><span id="DOCKING"></span>ドッキング


ドッキングと、アタッチ ドッキング定常 (非ポータブル) に移植可能なシステムのユーザーの操作を周辺機器と見なされます。 このようなアクションの例は次のとおりです。

-   ポート レプリケーターに接続します。
-   キーボードを統合した静止ドッキング アクセサリをアタッチします。 この場合は、ドッキング アクションとラップトップ/スレート変換の両方には、両方のインジケーターを実装する必要があります。

**注**   USB ベースのドッキング アクセサリが GPIO ドッキング インジケーターの実装の範囲外です。
