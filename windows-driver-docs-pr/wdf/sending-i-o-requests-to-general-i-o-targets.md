# <a name="sending-io-requests-to-general-io-targets"></a>一般 I/O ターゲットへの I/O 要求の送信





ドライバーは、同期または非同期 I/O の一般的なターゲットに I/O 要求を送信できます。

ドライバーは、同期的に I/O 要求を送信する場合、ドライバーのスレッドは、一度に 1 つの要求を送信します。 スレッドは、次の 1 つを送信する前に完了するには、各要求を待機します。 このプロセスは、I/O 要求を非同期的に送信するよりも簡単です。 ドライバーは、多くの要求を送信しない場合、および I/O 要求ごとに、ドライバーが待機中に、システムまたはデバイスのパフォーマンスが低下しない場合、I/O 要求を同期的に送信できます。

ドライバーでは、I/O 要求を非同期的に送信する場合、ドライバーのスレッドは、要求が先に送信された要求を完了を待つことがなく、送信できる状態にするとすぐに各要求を送信します。 ドライバーする必要があります多くの I/O 要求短い期間を処理する場合は、ドライバー、次の要求を送信する前に完了するには、各要求を待機する可能性があります許可できません。 それ以外の場合、データが失われるまたはドライバーのデバイスや、場合によっては、システム全体のパフォーマンスが低下する可能性があります。

フレームワークの I/O のターゲット オブジェクトには、ドライバーを呼び出すことができるメソッドの 2 つのセットが用意されています。 1 つに設定[I/O 要求を同期的に送信](sending-i-o-requests-synchronously.md)に他の設定と[I/O 要求を非同期的に送信](sending-i-o-requests-asynchronously.md)。

これらのメソッドごとに、要求オブジェクトとバッファー領域の一部を指定してください。 これらのメソッドを使用するには、ドライバーが、I/O キューのいずれかで受信した要求を転送するか、作成して、新しい要求を送信します。
