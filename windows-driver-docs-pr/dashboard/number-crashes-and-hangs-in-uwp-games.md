# <a name="number-of-crashes-and-hangs-in-uwp-games-normalized-by-usage-is-less-than-or-equal-to-the-baseline-goal"></a>UWP ゲームにおけるクラッシュとハングの数を使用量で正規化した値がベースライン目標と同じかそれより小さい

## <a name="description"></a>説明

ユーザーが UWP ゲームをプレイするとき、そのグラフィックス コンポーネントがビジュアル データを処理し、レンダリングされたビューを画面に表示します。 この測定値は、グラフィックス コンポーネントが原因で UWP ゲームがクラッシュしたりハングしたりする頻度を、ドライバーがインストールされているすべてのデバイスでのアプリケーションの実行時間の観点から監視するものです。 アプリケーションがクラッシュした場合、進行状況が失われる可能性があるうえ、再び使用できる状態になるまで、ユーザーはその回復を待機する必要があります。

## <a name="measure-attributes"></a>測定値の属性

|属性|Value|
|----|----|
|**オーディエンス**|エコシステム|
|**期間**|7 日間のスライディング ウィンドウ|
|**測定基準**|インスタンスの集計|
|**最小インスタンス**|100 時間 (UWP ゲームの実行時間)|
|**合格基準**|実行時のクラッシュが 1 年間に 5 回以下|
|**測定 ID**|14842478|

## <a name="calculation"></a>計算

1. この測定値は、**年単位に正規化された UWP ゲームの実行時間に対するクラッシュおよびハングの割合**として、7 日間のスライディング ウィンドウからのテレメトリを集計したものです。
2. "*UWP ゲームのクラッシュおよびハングの合計回数 = ドライバーがインストールされている各マシンで UWP ゲーム アプリケーションがクラッシュおよびハングした回数*"
3. "*UWP ゲームの総実行時間 = ドライバーがインストールされている各マシンにおける UWP ゲームの実行時間の合計*"
4. "*年間実行時間 = UWP ゲームの総実行時間 \* 60 (分) \* (時間) \* 24 (日) \* 365 (年)* "

### <a name="final-calculation"></a>最終的な計算

"*使用量で正規化した UWP ゲームのクラッシュ回数 = UWP ゲームのクラッシュおよびハングの合計回数 / 年間実行時間*"
