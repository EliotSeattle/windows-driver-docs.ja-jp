# <a name="firmware-weg-frequently-asked-questions-faq"></a>ファームウェア WEG:よく寄せられる質問 (FAQ)

次の FAQ は、まだ Windows 7 を使用しているコンピューターの台数が原因で発生しました。

追加の質問やコメントをに<SAUEFI@Microsoft.com>送信してください。

**Q:** Skylake に Windows 7 をインストールする場合は、どの構成を選択する必要がありますか。 ファームウェア CSM を UEFI ブート (config \#2) またはレガシブート MBR (config \#3) で有効にする必要がありますか。 レガシブートモードの場合、なぜでしょうか。

> **A:** 具体的には、Windows 7 の場合は Config \#2 を使用することをお勧めします。 CSM を有効にし、既定を UEFI ブートに設定する必要があります。これは、必要な場合にのみレガシブートにフォールバックします。 セキュアブートを無効にする必要があります。
>
> Windows 7 では、Config \# 1 をサポートできません。 Windows 7 では、上位のグラフィックドライバーが使用できない場合 (たとえば、バグチェック、スリープ遷移など)、CSM によって満たされた依存関係がある場合に、基本表示の Int10 サポートに依存しています。 部分的な CSM が配置されている場合 (有効)、Int10 サポートを提供している場合は、UEFI モードで Win7 を起動できます。 Int10 CSM を有効にしたまま、ファームウェアが既定で UEFI ブートモードになっている場合、Win7 は UEFI モードで正常にインストールされます。 また、Microsoft では、UEFI モードでインストールされている場合に、Windows 7 に対する TPM 2.0 のサポートを移植しています。 詳細については、次のサポート技術情報の記事を参照してください。[KB2920188](https://support.microsoft.com/help/2920188/update-to-add-support-for-tpm-2-0-in-windows-7-and-windows-server-2008)。
>
> Config \# 2 を使用することをお勧めします。レガシ MBR ブートモードは使用しないでください。

**Q:** CSM が有効になっている Windows 7 をインストールした場合、パーティションテーブル/ディスク構成はどのようになりますか?

> **A:** ファームウェアのブートデバイスの優先順位の既定値によって異なります。 CSM が有効になっていて、メモリに読み込まれている場合、インストールメディアがレガシ MBR ブートに既定で使用されることがあります。
>
> レガシブートモードで起動してインストールを実行すると、通常は2つのパーティションがあります (「diskpart.exe "list partition」の一覧を参照)。 最初のパーティションがアクティブになり、"システムパーティション" と表示されています。ファイル\\システムの種類は NTFS で、非表示のブートフォルダーがあります。 2番目のパーティション\\は、Windows が別の NTFS パーティションにインストールされる場所です。
>
> Windows が UEFI モードでインストールされている場合、インストール時に EFI システムパーティション (ESP) がファイルシステムの種類 Fat32、Microsoft 予約パーティション (MSR) として、NTFS を使用して OS/データパーティションとして作成されます。 ESP をマウントして表示すると、フォルダー \\EFI ブートフォルダー (フォルダーが非表示になっている可能性があります) が表示されます。
>
> Windows 7 のシステムファームウェアで CSM を有効にしたままにする必要があるため、UEFI ブートモードで Windows 7 をインストールするには、インストールメディア (CD/DVD/USB) の起動時に UEFI ブートモードを選択することが必要になる場合があります。
> ブートオプションと BIOS 構成に関する詳細については、「PC の製造元」を参照してください。

**Q:** CSM をレガシブートモード (config\#3) で有効にして Windows 7 をインストールした場合、アップグレードパスは何ですか。

> **A:** この構成のアップグレードパス (\#3) は、Windows 10 でもサポートされています。 ただし、セキュリティで保護されたブートを使用することはできません。 \# UEFI Config 1 でのみ使用できます。 システムを UEFI ブートに切り替えたい場合は、UEFI ブートモードを使用して、GPT ディスク用にパーティション分割されたディスクを使用して、元のオペレーティングシステムをフォーマットし、再インストールする必要があります。 Windows ストアまたはオンラインインストールからアップグレードする場合は、アップグレード (config\#2) のために windows 7 を起動する必要がない限り、この移行中に CSM を有効にする必要はありません。
> 純粋な UEFI 環境では構成\#1 に直接進むことをお勧めします (CSM が無効になっている場合)。これは、ファームウェアがサポートしている場合にセキュアブートを使用することです。

**Q:** レガシ BIOS ブート (config\#3) が有効になっている csm から、csm 対応 UEFI ブート (config\#2) およびバックアップにファームウェアを切り替えると、ユーザーに影響はありますか。

> **A:** これは、切り替えの方法によって異なります。 Config\#3 から config\#3 に変更した後、起動しようとすると、システムが起動しないことがあります。 この時点で config\#3 に戻す場合は、システム、ディスク、または OS を FURHTER に変更する必要はありません。 元の構成\#n を使用して元の OS に戻ることができます。 BIOS 設定を変更せずに、代わりに "その他" をアップグレードする*こと*をお勧めします。

**Q:** セキュアブートを\#有効にする\#には、config 2 から config 1 に変更します。

> **A:** 新しいシステムの場合は、post 2015 (Skylake)。システムで CSM が有効になっていて、セキュアブートが無効になっている場合 (相互に排他的である必要があります)、UEFI ブート方法を使用して OS が GPT ディスクにインストールされます。 Win10 + にアップグレードした後で、CSM をオフにし、ファームウェアでセキュアブートを有効にすることが簡単です (工場でセキュアブートがセットアップされており、ファームウェアでオプションになっている場合)。
>
> 変更を行った後、システムを起動できない場合は、システムが起動のために CSM のレガシ BIOS を使用して起動するように構成されている可能性が高くなります。 この場合、Win10 + インストールメディア、または元のオペレーティングシステム (Windows 7) のクリーンインストールが必要になります。 CSM が有効になっており、インストール/ディスクパーティションが UEFI ブートモードに設定されていることを確認します。これにより、後で UEFI ブートモードで Win10 にアップグレードできるようになります。
>
> メディアをインストールするには、必ず UEFI ブートを選択してください。 UEFI インストールを起動するには、ファームウェア構成で BIOS 設定の変更が必要になることがあります。
>
> ブートオプションと BIOS 構成に関する詳細については、「PC の製造元」を参照してください。

**Q:** ユーザーは Windows 7 を実行しています。 Win 10 にアップグレードする前に、BIOS/ファームウェアが UEFI に切り替わります。

> **A:** \[オペレーティングシステムが従来の BIOS ブートモードで以前にインストールされていることを前提としています。\]
>
> Windows 10 をインストールしているユーザーがインストール/セットアップを実行している場合。 Windows セットアップは、レガシ MBR BIOS ブートモードでディスクを検出し、それに応じてインストールを試行します。 結果はシステム、ブート方法、インストールメディア (カスタマイズされている場合) によって異なる場合があります。 ただし、ディスクフォーマットがこのブート方法でサポートされていないというメッセージが表示される場合があります。 このメッセージが表示された場合は、ディスクをフォーマットして新しいものをインストールするのが簡単です。 警告。ディスクをフォーマットすると、ディスク上のデータはすべて**失わ**れます。
>
> 推奨事項は、切り替え前の設定に戻すことです。 これは、既にインストールされているオペレーティングシステムで起動できるようにするためです。 保存するデータを外部ストレージの場所にバックアップします。 次に、ディスクの再プロビジョニングについて説明し (既存のパーティションのハードディスクをクリーニングし、UEFI モードでメディアを起動して、GPT ディスクへの新規インストールを実行します)。
>
> 細心構成\#2 のように CSM を有効のままにした場合は、インストールメディアが既定で MBR ブートになる可能性があります。 オペレーティングシステムがレガシ MBR ブートモードでインストールされている場合。 ディスクを再度再プロビジョニングして、オペレーティングシステムを UEFI ブートモードで GPT ディスクにインストールすることなく、セキュアブートを有効にすることはできません。 ブートオプション、BIOS 構成オプション、ファームウェアの更新、セキュアブートのオプションに関する詳細については、PC の製造元に問い合わせてください。

**Q:** UEFI は、MBR パーティション分割されたディスクからの起動をサポートしていますか。

> **A:** いいえ、MBR でパーティション分割されたディスクからの起動は、BIOS モードで起動している場合にのみサポートされます。 CSM が存在し、有効になっている場合は、CSM のレガシ BIOS ブート方法を利用してレガシブートがサポートされている必要があります。

**Q:** 32ビットとの依存関係の比較64-ビット UEFI?

> **A:** Microsoft では、32と64ビットの UEFI の両方をサポートしています。 OS と UEFI ファームウェアのビット数が一致している必要があります。 (たとえば、64ビットの UEFI ファームウェアがある場合は、64ビットの Windows OS をインストールする必要があります)。

**Q:** ファームウェアの設定をプログラムによって切り替えることはできますか。

> **A:** いいえ、ほとんどのファームウェア設定は標準化されていません。また、Microsoft では、ファームウェア設定をプログラムによって変更する手段を提供していません。 管理と構成に使用できるツールがある可能性があるため、詳細については、PC の製造元に問い合わせてください。

**Q:** Csm を有効にしたレガシ BIOS ブート (config\#3) から csm 対応 UEFI ブート (config\#2) への切り替えは、セキュアブートの動作に影響しますか。

> **A:** セキュアブートは、CSM が無効になっている UEFI モードでのみサポートされています。 セキュアブートが、お客様にシステムを出荷する前に工場フロアで事前に構成されていない場合は、使用可能なオプションがない可能性があります。 システムのセキュアブートが有効になっていても、現在の Windows 7 OS では無効になっている (ダウングレード権限を使用している) 場合は、CSM を無効にしてセキュアブートを有効にするのと同じくらい簡単です。 ただし、パーティションがクリーニングされ、UEFI ブートを GPT ディスクに設定していない場合は、 次に、セキュアブートを有効にする前に、ディスクをクリーニングし、UEFI ブート用のオペレーティングシステムを再インストールする必要があります。
>
> システムがセキュアブートをサポートしていることを確認するために、変更を行う前に、相手先ブランド供給 (OEM) に問い合わせてください。
>
> **メモ**ディスクをクリーニングすると、他のパーティションにある場合でも、そのディスク上のデータはすべて破棄されます。

**Q:** BitLocker および Microsoft 以外のディスク暗号化ツールに対するあらゆる依存関係

> **A:** はい。 UEFI/レガシ BIOS ブートを切り替えると、ブートが失敗する可能性があります。ファームウェア設定を既定値に戻すと、回復が可能になります。 Bitlocker を中断する場合、場合によっては、切り替えでドライブをワイプする必要があるか、またはファームウェア設定 (CSM/Secure boot など) を変更する必要があるかによって、Bitlocker または MS 以外の暗号化ツールを無効にすることをお勧めします。
>
> 構成と既定の詳細については、PC またはソフトウェアの製造元に問い合わせてください。

**Q:** BIOS が UEFI に切り替わると、セキュアブートはどのように構成されますか。

> **A:** 既定では、Win7 デバイスの場合、セキュアブートは実装されていません。 システムが Windows 8 以降に付属していた場合、これはロゴの要件であるため、システムにセキュアブートが構成されている可能性が非常に高くなります。 セキュアブートは、PC の製造元によって工場出荷時にのみ実装されます。 PC の製造元に問い合わせて、システムにセキュアブートをインストールできるかどうかを確認してください。 これをインストールするには、PC の製造元にシステムを送り返す必要がある場合があります。

**Q:** Windows 7 へのロールバック/復元にはどのような影響がありますか。

> **A:** ファームウェアの設定が変更されていない限り、元のオペレーティングシステムへのロールバックが機能します。 ファームウェアの設定が変更された場合は、それらを工場出荷時の既定値 (ロールバック前のもの) に戻します。 システムに含まれているロールバック方法または回復方法の種類については、「PC の製造元」を参照してください。

**Q:** WinRE にはどのような影響がありますか。

> **A:** WinRE は、インストールされているオペレーティングシステムと同じブートプロセスを実行します。 ファームウェアブートオプション (レガシ BIOS から UEFI) を変更した場合、オペレーティングシステムまたはディスク上の WinRE を起動しようとしても、エラーになります。 USB/CD/DVD で WinRE または WinPE を起動している場合は、別のブート方法が使用されます。

**Q:** 32ビットとの依存関係/影響64ビット OS はレガシブートから UEFI に移行します。ファームウェアが64ビット UEFI または32ビット UEFI の場合は、その逆になりますか。

> **A:** Windows UEFI ブートの制限事項の1つとして、インストールされている UEFI に一致する OS "ビット" をブートすることができます (質問7を参照)。 32ビットの UEFI がある場合は、32ビットの Windows のみをインストールすることができます。 64ビット Windows では、64ビットの UEFI または CSM が必要です。 ただし、システムに CSM が不足している場合は、Windows OS の 64 Win7 を起動することはできません。

**Q:** レガシ MBR ブートを使用して起動したシステムを、データに影響を与えることなく、即座に GPT ブートに変換できますか。

> **A:** No. Diskpart.exe などの Microsoft ツールを使用すると、データが失われる可能性があります。 MBR ブートと GPT ブートは、この変更を実装するためにディスクのすべてのデータを削除するディスクの変更を必要とする2つの異なるブートメカニズムです。 この変更を試みる前に、データをバックアップしてください。
