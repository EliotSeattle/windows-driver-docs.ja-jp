# <a name="smbios"></a>SMBIOS

SMBIOS 仕様では、システムに関連するデータ構造に入るデータ構造と情報が定義されています。 最新の SMBIOS 仕様を使用して、仕様に定義されている最新の変更について説明します。 次の表は、推奨される SMBIOS 設定と、これらのフィールドに含まれる情報の種類に関するガイダンスを示しています。 各システムに関連するデータがこれらのフィールドに設定されている場合、システム管理者は、これらのシステムをリモートで識別して管理することができます。 コンピューターのハードウェア Id (CHIDs) は、これらの表の値を使用して生成されます。これらを設定する場合は注意が必要です。

デバイス情報をより明確に識別できるように、SMBIOS に均一性を追加するには、SMBIOS フィールドを設定する際のガイダンスとして次のことをお勧めします。 以下の SMBIOS データも収集され、さまざまな容量で使用されます。 これらのフィールドに入力するデータは、BIOS/ファームウェアベンダーが提供するツールを使用してデータを設定する前に、詳細に計画する必要があります。 CHID をターゲットとするために生成されるハッシュは、これらのフィールドを設定するデータに基づいています。

この情報は、「 [Windows 10 Driver Publishing Workflow](https://download.microsoft.com/download/B/A/8/BA89DCE0-DB25-4425-9EFF-1037E0BA06F9/windows10_driver_publishing_workflow.docx)」に記載されているものと似ていますが、次の表に、いくつかのフィールドの詳細レベルを示しています。これにより、特異性のレベルが向上します。
