# <a name="desktop-cosaapn-database-settings"></a>デスクトップ COSA/APN データベースの設定

このトピックでは、desktop COSA および APN データベース (apndatabase .xml) で使用できる設定について説明します。

デスクトップ COSA/APN データベースの送信プロセスの詳細については、「[デスクトップ COSA/apn データベースの送信の計画](planning-your-desktop-cosa-apn-database-submission.md)」を参照してください。

COSA の詳細については、 [Cosa の概要](cosa-overview.md)に関するトピックを参照してください。

APN データベースの詳細については、「 [apn データベースの概要](apn-database-overview.md)」を参照してください。

## <a name="apn-database-only-settings"></a>APN データベースのみの設定

スプレッドシートの次の設定は、APN データベースのみに適用されます。 これらのエントリは、windows 8、Windows 8.1、または windows 10 バージョン1703より前のバージョンの Windows 10 を対象としている場合に使用されます。

| 設定名 | 説明 | 省略可能または必須 | 説明 |
| --- | --- | --- | --- |
| CDMA の ProviderID | デバイスによって報告された CDMA プロバイダー ID (SID とも呼ばれます) と一致する5桁の数字。 | ［オプション］ | |
| CDMA ProviderName | デバイスによって報告された CDMA プロバイダー名と一致する必要がある、36文字以内の文字列。 この設定では大文字と小文字が区別されます。 | ［オプション］ | |
| CertIssuerName | オペレーターの XML プロビジョニングに使用される署名証明書の証明書発行者名。 | ［オプション］ | 指定する場合は、証明書のサブジェクト名とキャリア GUID も指定する必要があります。 |
| CertSubjectName | オペレーターの XML プロビジョニングに使用される署名証明書の証明書のサブジェクト名。 | ［オプション］ | 指定する場合は、証明書の発行者名とキャリア GUID も指定する必要があります。 |
| 通信事業者の GUID | 将来の XML プロビジョニングパッケージで使用される自己割り当て GUID。 | ［オプション］ | 指定する場合は、証明書のサブジェクト名と証明書発行者名も指定する必要があります。 |
| GSM ProviderName | デバイスによって報告された GSM プロバイダー名と一致する必要がある、36文字以内の文字列。 この設定では大文字と小文字が区別されます。 | ［オプション］ | この設定は、Windows 10 バージョン1703より前の Windows 8.1 およびバージョンの Windows 10 でのみサポートされています。 |
| 購入 | APN がプロビジョニングまたは購入されているかどうかを示す、yes または no 値。 | 必須 | 次の値を使用できます。 <ul><li>**Y** – APN がプロビジョニングまたは購入されている場合</li><li>**N** -APN がプロビジョニングまたは購入されていない場合</li></ul> <p>**購入フラグ**の設定が**Y**の場合、 **Connect フラグ**の設定は**N**にする必要があります。</p> |
| 接続 | APN がプロビジョニングまたは購入されているかどうかを示す、yes または no 値。 | 必須 | 次の値を使用できます。 <ul><li>**Y** – APN がプロビジョニングまたは購入されている場合</li><li>**N** -APN がプロビジョニングまたは購入されていない場合</li></ul> <p>**接続フラグ**の設定が**Y**の場合、**購入フラグ**の設定は**N**である必要があります。</p> |

| 国/地域 | APN エントリの国または地域。 | 必須 | このエントリは、Windows が特定の国または地域を参照する方法に合わせて変更される可能性があります。 |  Windows 10 Version 1703 |
| 演算子 | オペレーターの名前。 このフィールドに国または地域を含める必要はありません。 | 必須 | APN エントリの更新を送信するたびに、必ず同じスペルと大文字小文字が使用されるようにしてください。 | Windows 10 Version 1703 |

|  IMSI 範囲-終了  | 数値の先頭に MCC + MNC を含む15桁の数字。 | ［オプション］ | この数値は99で終わる必要があります。 <p>この設定と**Imsi 範囲開始**設定を空白のままにして、 **MCC**と**mnc**エントリを指定すると、MCC + mnc 範囲全体が対象になります。</p> | Windows 10 Version 1703 |
| ICCID の範囲-開始 |  89 (ICCID issuer 識別子番号) で始まる 19 ~ 20 桁の数値。 | ［オプション］ | この数値は00で終了する必要があります。  | Windows 10 Version 1703 |
| ICCID 範囲-終了  |  89 (ICCID issuer 識別子番号) で始まる 19 ~ 20 桁の数値。 | ［オプション］ | この数値は99で終わる必要があります。  | Windows 10 Version 1703 |
| AccountExperienceURL |  ユーザーがアクティブなプランを持っておらず、ネットワークへの接続を試行する場合に、Windows 接続マネージャーによって使用されます。  | ［オプション］ | プランの取得エクスペリエンスの向上に役立ちます。 <p>Appid と AccountExperienceURL の**両方**が指定されている場合、appid は優先順位が高くなり、UX は AccountExperienceURL を表示し**ません**。</p> | Windows 10 Version 1703 |
| FriendlyName | この APN エントリの名前。サブスクライバーにとってわかりやすくわかりやすい名前です。 | ［オプション］ | Windows がネットワークに自動的に接続できない場合に、Windows 接続マネージャーに表示されます。 | Windows 10 Version 1703 |
| AccessString | GSM ネットワークの場合、これは data.contoso.com などの APN です。 CDMA ネットワークの場合、これは #777 などの特別なダイヤルコードや example@contoso.comなどのネットワークアクセス識別子を含むアクセス文字列です。 | 必須 |  | Windows 10 Version 1703 |

| AuthProtocol | パケットデータプロトコル (PDP) のコンテキストをアクティブ化するために使用する認証プロトコルを指定します。 | ［オプション］ | 次の値を使用できます。 <ul><li>**なし**–認証プロトコルは必要ありません</li><li>**Pap** – pap 認証が必要です。</li><li>**Chap** : chap 認証が必要です。</li><li>**Mschapv2** – mschapv2 は認証が必要です。</li></ul> <p>この設定は、Windows 10 バージョン1703より前の Windows 8.1 およびバージョンの Windows 10 でのみサポートされています。</p> | Windows 10 Version 1703 |

| AutoConnectOrder | Windows は、オペレーターによって提供された APNs への接続を試行し、デバイスが正常にモバイルネットワークに接続するまで、APN データベースまたは COSA で "自動接続" としてマークされます。 すべての自動接続の試行が失敗した場合は、ユーザーが APN を選択したり、カスタム APN を入力したりすることを許可するプロンプトが表示されます。 | ［オプション］ | 1つのオペレーターに対して複数のアクセス文字列がある場合、この設定は1から始める必要があります。 これは、ユーザーが接続しようとしたときに、Windows が IMSI 範囲、ICCID range、CDMA プロバイダー ID、または CDMA プロバイダー名を共有する複数の APN エントリを試すために必要です。 | Windows 10 Version 1809 |

| SPN | サービスプロバイダー名 (SPN) の識別子文字列 | ［オプション］ | MO/MVNO のネットワークを特定するのに役立ちます。 空白のままにすると、既定値は空の文字列になり、何も実行されません。 | Windows 10 Version 1703 |
| AlwaysOn | 接続が常にオンであるかどうかを示します。 | 必須 | 空白のままにすると、既定値は "true" になります。 | Windows 10 Version 1703 |
| PurposeGroups | 目的の値を表す Guid のコンマ区切りリストを使用して、接続の目的を指定する文字列。 | 必須 | 次の目的値を使用できます。 <ul><li>**インフォメーション**</li><li>**MM**</li><li>**IM**</li><li>**SUPL**</li><li>**ご**</li><li>**レベル**</li><li>**アプリケーション**</li><li>**eSIM のプロビジョニング**</li></ul> <p> 空白のままにすると、既定では "Internet" になります。</p> | Windows 10 Version 1703 |
| | | ［オプション］ | 次のオプションの目的グループの値を使用できます。 <ul><li>**LTE アタッチ** </li></ul> 添付情報は通常、基になるモデムのハードウェアによって直接管理されるため、通常は必要ありません。 ただし、モデムのハードウェアに LTE 接続の APN 値がなく、ネットワークがそれを UE-V に割り当てることができない場合、必要に応じて、ユーザーが手動で値を入力しなくても、UE-V をアタッチできるように LTE アタッチ情報を提供できます。 | Windows 10 バージョン 1903 |
| IPType | 接続のネットワークプロトコルを指定する文字列。 | ［オプション］ | 次の値を使用できます。 <ul><li>IPv4</li><li>IPv6</li><li>IPv4v6</li></ul> <p>空白のままにすると、既定で "IPv4" になります。</p> | Windows 10 Version 1703 |
| BrandingName | モバイルブロードバンドデバイスは、通常、windows 接続マネージャーに表示されるオペレーター名を提供します。 この名前をオーバーライドするには、メタデータでカスタム名を指定します。 | ［オプション］ | 空白のままにすると、既定値は空の文字列になり、何も実行されません。 | Windows 10 バージョン 1709 |
