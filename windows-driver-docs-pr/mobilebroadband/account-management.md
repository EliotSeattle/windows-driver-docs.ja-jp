# <a name="account-management"></a>アカウント管理


ユーザーは、サブスクリプションを購入した後は、次のタスク実行できます。

-   **現在のデータ使用状況を表示**ユーザーが現在のデータの使用状況を表示し、データの使用量を適切な決定を行うために、請求サイクル (またはセッションの終了日) を理解します。

-   **アカウントの設定を管理する**ユーザーを表示して、支払とアカウントの詳細 (パスワード、電子メール アドレス、自動支払情報など) を安全に管理します。

-   **料金を支払います**ユーザーは、UWP アプリを使用して、定期的または 1 回の請求書を支払うことができます。

アカウント管理機能は、演算子で、サブスクライバーのリレーションシップを表示します。 これを使用して、競合他社のサービスからサービスを識別するブランド化されたエクスペリエンスを作成することができます。

設計に関する考慮事項を以下に示します。

-   **データの使用状況とバックエンドのローカル情報の組み合わせを行う**可能な限りバック エンド サーバーの負荷を減らすためには、Windows はバック エンドのデータの使用状況を結合するために使用できるローカル データ使用量 API を提供します。 定期的に、バック エンドから使用状況情報を取得してローカル データの使用状況を関連付けることができます。

-   **データの使用状況で Windows を定期的に更新**Windows 10 が従量制課金接続で適切に動作設計されています。 Windows および UWP アプリは、モバイル ブロード バンド ネットワークを使用して、必要なトラフィックのための重要なネットワーク容量を節約できます。 正確になるし、アプリケーション (たとえば、データの制限と使用状況) の詳細については、Windows が定期的に正しい情報を提供することに依存します。 アプリでは、データ使用量 Api を使用して、この情報を設定できます。
