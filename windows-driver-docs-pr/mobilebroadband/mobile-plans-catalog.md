# <a name="mobile-plans-operator-catalog"></a>モバイルプランオペレーターカタログ

## <a name="overview"></a>概要

モバイルプランアプリを使用すると、ユーザーはモバイルオペレーターの一覧を表示して選択できます。 この記事では、[プロバイダーの選択] ページで、モバイルオペレーターの表示に適用される原則を定義します。

> [!NOTE]
> [プロバイダーの選択] ページは、eSIM が有効になっている Windows 10 Pc でのみ使用できます。 レガシの物理 UICCs ではモバイルオペレーターの SIM プロファイルを変更できないため、デバイスにアクティブな eSIM がない場合、[プロバイダーの選択] ページは使用できません。

![[プロバイダーの選択] ページ](images/select_provider_page.png)

## <a name="list-of-mobile-operators"></a>携帯電話事業の一覧

- ユーザーは、モバイルプランで有効になっている携帯電話オペレーターの一覧を表示して、特定の国でデータプランを販売することができます。
- デバイスの物理的な場所は、国を決定するために使用されます。 デバイスが携帯ネットワークを使用して接続されている場合、その国を特定するためにネットワーク ID が使用されます。 デバイスが Wi-fi 経由で接続されている場合、その国は逆引き IP によって決定されます。
- ユーザーに表示される携帯電話の一覧は、携帯電話会社のブランド名でアルファベット順に表示されます。

## <a name="mobile-operator-branding"></a>携帯電話会社
- カタログに示されている携帯電話会社は、その会社の携帯電話サービスに一般的に関連付けられているブランド名と商標を使用する必要があります。 これは、ユーザーがサービスプロバイダーの身元を明確に理解できるようにするためです。
- Windows PC OEM に関連付けられているブランド要素を含む携帯電話会社のブランド名は、これらの OEM デバイスにのみ表示されます。 これは、oem ブランド要素が別の OEM デバイスのユーザーに表示されないようにするためです。 携帯電話会社または OEM は、OEM ブランドの携帯電話事業者が使用できるようにするデバイスの製造元とモデルを提供する必要があります ([Windows システム情報] ダイアログボックスに表示されます)。

## <a name="device-filtering"></a>デバイスのフィルター処理
- 携帯電話会社との互換性がないデバイス (モデム、eSIM ハードウェア、または、または、またはの eSIM) がある場合、携帯電話会社は、そのデバイスに表示されている携帯電話の一覧からフィルター処理するように要求できます。 これは、エンドユーザーが自分のデバイスで使用できないオペレーターからデータプランを取得しようとしないようにするためです。 フィルター処理を有効にするには、デバイスの製造元とモデル (Windows システム情報ダイアログで検出されたもの)、または eSIM の eID が携帯電話会社から提供されている必要があります。
