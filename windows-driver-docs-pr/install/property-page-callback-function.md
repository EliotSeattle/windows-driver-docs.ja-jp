# <a name="property-page-callback-function"></a>プロパティ ページのコールバック関数





プロバイダーは、そのデバイスまたはデバイス クラスのプロパティ ページを作成するときに、コールバック関数へのポインターを提供します。 コールバック関数では、プロパティ ページが作成されたとき、もう一度破棄する場合と呼ばれる 1 つの時間です。

コールバックは、 **PropSheetPageProc** Windows SDK のドキュメントで説明されている関数。 この関数は、PSPCB_CREATE と PSPCB_RELEASE アクションを処理できる必要があります。

プロパティ ページを作成するとき、PSPCB_CREATE メッセージでコールバックが呼び出されます。 このメッセージに応答して、コールバックは、ページに関連付けられているデータのメモリを割り当てることができます。 関数が返す必要があります**TRUE** 、ページの作成を続行または**FALSE**場合は、ページを作成しないようにします。

ユーザーがクリックすると、デバイスのプロパティ ページは破棄されます **[ok]** または**キャンセル**ページのダイアログ ボックスまたは数回のクリックで**アンインストール**上、 **ドライバー**タブ。

プロパティ ページが破棄されると、PSPCB_RELEASE メッセージでコールバックが呼び出されます。 関数は、プロパティ ページの作成時に割り当てられたすべてのデータを解放する必要があります。 通常、この層で参照されるデータを解放する、 **lParam** PROPSHEETPAGE 構造体のメンバー。 ページが破棄されるときに、戻り値は無視されます。
